# v5.0 产品成熟化执行计划

> 产出人：斯维因 | 时间：2026-03-26
> 状态：李琪已确认方向，待排期执行

---

## 目标
让产品从"功能堆叠"升级为"成熟产品形态"，用户双击 exe 就能感受到这是一个完整、专业的产品。

---

## 任务清单（按优先级排序）

### P0：插件中心 + 分层导航
> 当前最大的"不成熟感"来源：19 个插件平铺在导航栏

**任务 1：插件分层标记**
- manifest.json 新增 `layer`（`core` / `builtin` / `extension`）和 `category` 字段
- 19 个插件逐个标记
- 工作量：小（0.5 天）

**任务 2：插件中心页面**
- 新建"🧩 插件中心"页面，卡片式展示所有扩展插件
- 分类筛选：AI 智能 / 数据治理 / 报表审计 / 流程协作 / 外部集成 / 资源市场
- 启用/停用开关（管理员）
- 插件状态持久化（platform_db 新增 plugin_status 表）
- 预留"需授权"锁状态（对接统一架构）
- 工作量：中（1.5 天）

**任务 3：导航栏分组渲染**
- 底座功能：常驻一级导航
- 基础工具（7 个）：分组显示
- 扩展功能：启用的插件动态插入，未启用的不显示
- 工作量：中（1 天）

### P1：启动加载页
> 用户双击启动后不是白屏等待，而是有品牌感的 loading

**任务 4：启动 Loading 页**
- 服务启动过程中显示加载页面（品牌 logo + "启动中..." + 百分比进度条）
- 进度阶段：初始化数据库 → 加载插件 → 启动服务 → 就绪
- 后端新增 `/api/startup-progress` 接口返回当前启动阶段和进度
- 前端 loading 页面轮询进度，100% 后跳转登录页
- 工作量：中（1 天）

### P2：离线可用性优化
> 明确告知用户哪些功能离线可用

**任务 5：离线/在线状态提示**
- 启动时检测网络连通性
- AI 功能入口在离线时显示"需联网"提示而非报错
- 系统设置页显示当前网络状态
- 工作量：小（0.5 天）

## 验收标准
1. 导航栏分三组（底座 / 基础工具 / 扩展功能），不再平铺 19 个插件
2. 插件中心页面可浏览所有扩展插件，管理员可启用/停用
3. 启用的扩展插件自动出现在导航栏，停用的消失
4. 启动时有 loading 页面，带进度百分比
5. AI 功能离线时有明确提示
