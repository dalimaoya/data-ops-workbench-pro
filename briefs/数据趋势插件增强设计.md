## 增强设计

### 一、三级维度结构

```
数据趋势
├── 全局概览（默认页）
├── 按数据库查看
└── 按数据表查看
```

#### 1. 全局概览（Overview）
- **顶部统计卡片**：总操作数、回写数、导出数、导入数（按时间范围）
- **全局趋势图**：折线图展示每日操作量
- **数据库活跃度排名**：柱状图/表格，按数据库维度聚合操作数
- **数据表活跃度 Top 10**：柱状图/表格

#### 2. 按数据库查看
- **顶部**：数据库选择器（下拉，默认选第一个健康数据库）
- **统计卡片**：该库下纳管表数、总操作数、回写/导出/导入分项
- **趋势图**：该库每日操作量折线图
- **表活跃度排名**：该库下各表的操作数排名
- **操作类型分布**：饼图（回写/导出/导入/编辑/删除）

#### 3. 按数据表查看
- **顶部**：数据库 → 表 级联选择
- **统计卡片**：总操作数、最近一次操作时间、最活跃操作人
- **趋势图**：该表每日操作量折线图（可叠加显示操作类型）
- **热门字段**：被修改最多的字段 Top 10
- **变更明细时间线**：最近 N 条操作记录，时间线展示

### 二、时间维度

所有页面统一的时间筛选器（页面右上角）：

| 选项 | 说明 |
|------|------|
| 今天 | 当天 |
| 最近 7 天 | 默认选中 |
| 最近 30 天 | |
| 最近 90 天 | |
| 最近 365 天 | |
| 自定义范围 | 日期区间选择器 |

时间切换后，页面所有图表和统计实时更新。

### 三、后端 API 调整

#### 修复：日志 target_id 关联
- 排查所有 `log_operation` 调用点，确保 target_id 正确写入表 ID
- 对于数据库维度聚合：通过 table_config.datasource_id 关联

#### 新增/调整 API

```
GET /api/data-trend/overview
  ?days=7|30|90|365
  &start_date=YYYY-MM-DD（自定义范围）
  &end_date=YYYY-MM-DD

GET /api/data-trend/datasource/{ds_id}
  ?days=7
  返回：该库下的趋势、表排名、操作分布

GET /api/data-trend/table/{table_id}/history
  ?days=7
  返回：该表的趋势、热门字段、变更时间线（已有，需增强）
```

### 四、前端页面结构

```
DataTrendPage.tsx
├── Tabs: 全局概览 | 按数据库 | 按数据表
├── 右上角: 时间筛选器（统一组件）
├── OverviewTab.tsx  — 全局统计 + 趋势图 + 排名
├── DatasourceTab.tsx — 选库 + 库级趋势 + 表排名 + 饼图
└── TableTab.tsx — 选库→选表 + 表级趋势 + 热门字段 + 时间线
```

图表使用纯 Canvas 绘制（和首页趋势图保持一致，不引入新依赖）。
