# v3.5 — 模板安全加固

## 版本目标
防止用户在 Excel 模板中误操作（改列头、删字段列、改主键），导致导入回写出问题。

---

## 功能 1：模板 Excel 保护机制

### 1.1 表头行保护
- 模板第一行（列头）锁定，不可修改、不可删除
- 用 openpyxl 的 worksheet.protection 实现

### 1.2 列结构保护
- 导出几个字段就锁定几列，不允许用户删除列或插入新列
- 保护粒度：列级别

### 1.3 主键列保护
- 已有数据行的主键单元格设为只读
- 模板底部预留 50 行空白区域（主键列不锁定），供用户填写新增数据
- 新增行的 INSERT 回写能力在 v4.0 落地，但模板结构要提前兼容

### 1.4 工作表保护
- 使用 Excel 工作表保护功能（带密码）
- 仅允许编辑"可编辑字段"的数据单元格
- 保护密码不暴露给用户，系统内部管理

### 1.5 保护机制的视觉提示
- 只读单元格背景色设为浅灰（#F0F0F0）
- 可编辑单元格保持白色
- 表头行加粗 + 蓝色背景
- 模板底部空白区域加浅黄色背景提示"新增数据区"

## 功能 2：导入校验增强

### 2.1 列数校验
- 上传文件的列数与模板定义的字段数不一致时，阻断导入
- 报错信息：「上传文件包含 X 列，模板定义 Y 列，请使用最新模板」

### 2.2 列名校验
- 逐列比对列头名称与模板定义
- 不一致时阻断导入
- 报错信息：「第 N 列名称不匹配：期望 'XXX'，实际 'YYY'」

### 2.3 主键不可变校验
- 对比导入数据中主键值与原始数据
- 如果主键值被修改，报错并阻断
- 报错信息：「第 N 行主键值被修改：原值 'XXX'，新值 'YYY'，主键不允许修改」

### 2.4 数据类型校验增强
- 数值字段：检查是否为合法数字
- 日期字段：检查格式是否符合预期
- 非空字段：检查必填字段是否为空
- 报错时精确到行号 + 列名

## 验收标准

### 模板保护
1. ✅ 下载的模板，列头行不可编辑（Excel 打开后尝试修改会提示受保护）
2. ✅ 不可删除列或插入新列
3. ✅ 已有数据行的主键列不可编辑
4. ✅ 可编辑字段的数据单元格可以正常编辑
5. ✅ 模板底部有 50 行空白预留区域
6. ✅ 只读/可编辑单元格有明确的颜色区分

### 导入校验
7. ✅ 列数不匹配时阻断，提示具体差异
8. ✅ 列名不匹配时阻断，提示具体位置
9. ✅ 主键被修改时阻断，提示具体行和值
10. ✅ 类型校验错误精确到行号 + 列名
11. ✅ 校验通过后才允许继续导入流程
