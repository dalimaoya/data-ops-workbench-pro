# 产品补丁需求：INSERT 新增行数限制放开

> 提出人：斯维因 | 时间：2026-03-26 | 优先级：P0

## 问题
v1.2 模板安全加固时，模板底部预留了固定 50 行空白行。v3.9 上线 INSERT 回写后，这 50 行上限成为用户批量导入新数据的瓶颈——如果用户需要一次新增超过 50 行数据，当前模板无法满足。

## 产品要求

### 方案：动态空白行 + 用户可自行追加
1. **模板导出时**：空白行数从固定 50 行改为可配置，默认值提升到 200 行
2. **取消硬性行数上限**：导入校验时，不以模板预留行数为上限，只要用户在模板底部追加了合法数据行（主键不在数据库中），一律识别为新增行
3. **字段配置页**：纳管表配置中新增「模板预留空白行数」字段，管理员可按表设置（默认 200，范围 50-10000）

### 验收标准
- [ ] 默认模板空白行数为 200（而非 50）
- [ ] 用户在模板中追加超出预留行数的数据，导入校验仍能正确识别新增行
- [ ] 纳管表配置页可设置预留行数
- [ ] Excel 保护机制不阻止用户在空白区域以外继续追加行

## 影响评估
- 涉及模块：模板导出、导入校验、纳管表配置
- 不影响 UPDATE / DELETE 链路
- 向下兼容：旧模板（50 行）的导入不受影响
