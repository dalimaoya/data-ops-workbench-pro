# v3.4.1 — Windows 启动器 + 打包修复

## Bug 修复

### start.bat 路径不匹配
- start.bat 中检测的是 `server\app\app.exe`
- 但 Nuitka 编译输出的是 `dataops-server.exe`（或 `dataops-server`）
- 需要修复 start.bat 中的路径，同时修复 CI workflow 的打包组装步骤确保路径一致

## 新需求：Windows 桌面启动器（exe）

### 用户期望
- 不要 .bat 文件启动
- 要一个 .exe 程序，双击启动
- 后台运行服务，不弹命令行黑窗口
- 有一个简版弹窗/托盘图标，显示运行信息

### 启动器功能
1. **系统托盘图标**（右下角小图标）
2. **弹窗/主界面**显示：
   - 当前版本号（如 v3.4.1）
   - 服务运行状态（运行中 / 已停止）
   - 服务地址（http://localhost:8580）
   - 「打开控制台」按钮 → 拉起默认浏览器打开 http://localhost:8580
   - 「启动服务」/「停止服务」按钮
   - 版本检测（可选，检查 GitHub Releases 是否有新版本）
3. **后台运行**：启动器启动后，dataops-server 在后台运行，不显示命令行窗口
4. **关闭行为**：关闭弹窗最小化到托盘，右键托盘可退出（退出时停止服务）

### 技术方案建议
- 用 Python + tkinter（打包时一起编译进 exe）
- 或用 Python + PyQt/PySide（更美观但依赖更重）
- 或用纯 C/Go 写一个轻量启动器（独立于主服务）
- 推荐用 tkinter，因为已有 Python 环境，打包最方便

### 启动器打包
- 启动器本身也用 Nuitka 编译成单独的 exe（launcher.exe）
- 最终 Windows 发布包结构：
```
data-ops-workbench/
├── 数据运维工作台.exe        # 启动器（双击运行）
├── server/
│   └── app/
│       └── dataops-server.exe  # 后端服务
├── data/
├── backups/
├── logs/
├── start.bat                   # 保留作为备用启动方式
└── README.md
```

### 可选增强：Inno Setup 安装包
- 用 Inno Setup 打成 .exe 安装包
- 安装时自动创建桌面快捷方式和开始菜单项
- 这个可以后续做，先把启动器搞定
