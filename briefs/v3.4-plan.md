# v3.4 版本计划 — Windows 打包 + GitHub Actions CI

## 版本目标
1. 配置 GitHub Actions，实现打 tag 自动编译 Linux + Windows 版本并发布到 Releases
2. 产出 Windows 版本的 Nuitka 编译包（exe）

## 任务拆解

### 任务 1：GitHub Actions Workflow（优先）

在私有仓库 `data-ops-workbench-pro` 中配置 `.github/workflows/release.yml`：

触发条件：推送 `v*` 格式的 tag（如 `v3.4.0`）

Job 1 — Linux 编译：
- runner: ubuntu-latest
- 安装 Python 3.11 + gcc + ccache + pnpm
- 构建前端
- Nuitka --standalone --jobs=4 编译后端
- 打包为 data-ops-workbench-vX.Y.Z-linux-x64.tar.gz
- 上传到 GitHub Release

Job 2 — Windows 编译：
- runner: windows-latest
- 安装 Python 3.11 + MinGW(或MSVC) + pnpm
- 构建前端
- Nuitka --standalone --jobs=4 --mingw64 编译后端
- 可选：用 Inno Setup 打成 exe 安装包
- 打包为 data-ops-workbench-vX.Y.Z-win-x64.zip（或 .exe）
- 上传到 GitHub Release

### 任务 2：build-nuitka.sh 适配

- 支持 --platform 参数区分 linux/windows
- Windows 路径兼容
- start.bat 适配 Nuitka 产物

### 验收标准
1. ✅ 推送 v3.4.0 tag 后 GitHub Actions 自动触发
