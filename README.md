## 🎯 这是什么？

数据运维工作台是一个**轻量级的数据安全修订平台**，专为政企项目交付和运维现场设计。让普通运维和业务人员在完全不接触数据库客户端的前提下，通过「平台模板 + 校验预览 + 安全回写」的标准流程，完成结果数据表的日常维护。

它不是数据库开发工具，不是 BI 分析平台，也不是传统的数据填报系统——它只聚焦一件事：**让改数据这件事变得安全、可控、可追溯。**

### ✨ 核心亮点

- 🔒 **安全回写** — 写前校验、差异预览、自动备份、一键回退，四重保障
- 📋 **平台模板** — 所有修订通过受控模板完成，杜绝任意 Excel 和手写 SQL
- 🌐 **7 种数据库** — MySQL / PostgreSQL / SQL Server / Oracle / 达梦(DM) / 人大金仓(KingbaseES) / SQLite
- 🌍 **中英文双语** — 完整的国际化支持，一键切换
- 👥 **多角色权限** — 管理员 / 操作员 / 只读，三角色分权 + 数据源级权限隔离
- ✅ **审批流** — 可选的回写审批机制，操作员提交 → 管理员审核 → 才执行
- 🔔 **通知中心** — 回写完成、审批状态变更等关键事件实时通知
- 📦 **零依赖部署** — 拷贝文件夹即可运行，不需要安装 Python / Node / 数据库
- 🖥️ **跨平台** — Linux / Windows / macOS / Docker 均可运行

## 🔧 解决什么问题？

在项目交付和运维现场，修改数据库中的结果数据是高频操作，但现有方式普遍存在风险：

| 现状痛点 | 具体表现 | 工作台方案 |
|---|---|---|
| 🔓 直接改库风险高 | 用 Navicat 写 SQL，一条 UPDATE 写错就不可逆 | 平台模板 + 校验预览 + 写前备份 |
| 📎 线下 Excel 低效 | 导出 → 微信传 → 人工粘贴回库，版本混乱 | 平台受控模板，嵌入元信息防篡改 |
| 🗣️ 口头改数无留痕 | 谁改的、改了什么、改了多少，事后无从查证 | 全程操作日志 + 逐字段变更明细 |
| 🔧 临时脚本不可见 | 运维写临时脚本处理，过程不透明 | 标准化流程 + 版本回退 |
| 🚫 普通人员无法操作 | 每次改数据都要找研发 | 研发一次配置，用户反复使用 |

## 🏗️ 设计原则

### 安全第一
- **写前必校验** — 数据类型、必填项、枚举值、字段完整性全面检查
- **写前必预览** — 逐行逐字段展示差异，确认后才执行
- **写前必备份** — 每次写入前自动全表备份，保留最近 3 个版本
- **写后可回退** — 出问题一键回退到任意备份版本

### 平台模板机制
- 所有修订必须通过平台导出的标准模板完成
- 模板嵌入隐藏元信息（版本、表标识），非平台模板无法上传
- Excel 工作表保护：表头锁定、主键锁定、仅可编辑区域允许修改

### 最小权限
- 三角色分权：管理员配置、操作员执行、只读查看
- 数据源级权限隔离：不同用户只看到授权的数据源
- 普通用户全程不接触数据库连接信息
- 可选审批流：操作员的回写需管理员审核

## 🗺️ 主链路

### 管理员配置链路（一次性）

```
新建数据源 → 测试连接 → 选择纳管表 → 自动拉取字段
→ 配置字段语义与编辑规则 → 指定主键 → 保存
```

### 用户操作主链路（日常重复）

```
选择目标表 → 浏览当前数据 → 导出平台模板 → 本地 Excel 修改
→ 上传模板 → 系统校验 → 差异预览 → 确认回写
→ 备份+写入 → 结果反馈 → 日志留痕
```

### 异常处理

- **校验失败** → 导出错误明细 → 本地修正 → 重新上传（或一键重新校验）
- **表结构变化** → 自动检测并阻断 → 管理员同步字段 → 重新配置
- **需要回退** → 版本回退页 → 选择版本 → 确认回退

## 📦 产品形态

**交付物是一个完整的文件夹**，拷贝到目标机器即可运行：

- Linux / macOS → 运行 `./start.sh`

## 🚀 快速启动

### 方式一：源码运行

**前置条件：** Python 3.9+ / Node.js 18+

```bash
# Linux / macOS
chmod +x start.sh
./start.sh

# Windows
start.bat
```

### 方式二：Docker

```bash
# docker-compose 一键启动
docker-compose up -d

# 或手动构建
docker build -t data-ops-workbench .
docker run -d -p 8580:8580 \
  -v ./data:/app/data \
  -v ./backups:/app/backups \
  -v ./logs:/app/logs \
  data-ops-workbench
```

### 方式三：独立打包（推荐生产使用）

```bash
# Linux
chmod +x build.sh && ./build.sh

# Windows（必须在 Windows 环境下执行）
build.bat
```

打包产物结构：
```
dist/data-ops-workbench/
├── start.bat / start.sh    # 启动脚本
├── server/app/             # 可执行文件（含所有依赖）
├── data/                   # 运行时数据
├── backups/                # 备份目录
├── logs/                   # 日志目录
└── README.md
```

使用：解压 → 运行启动脚本 → 浏览器访问，零环境依赖。

### 启动后

| 项目 | 地址 |
|---|---|
| 系统首页 | http://localhost:8580 |
| API 文档 | http://localhost:8580/docs |
| 默认账号 | `admin` / `dalimaoya` |

## 🔐 安全说明

| 安全措施 | 说明 |
|---|---|
| 登录验证码 | 图形验证码防暴力破解，5 分钟有效，一次性使用 |
| JWT 认证 | 24 小时有效期，支持自定义密钥 |
| 密码哈希 | bcrypt 加密存储，不存明文 |
| 权限校验 | 前端菜单隔离 + 后端 API 层角色拦截 |
| 数据源密码 | 加密存储，前端不回显 |
| 操作审计 | 所有操作记录日志，含操作人、时间、内容 |
| 写前备份 | 每次回写前自动全表备份 |
| 模板防篡改 | 嵌入隐藏元信息，非平台模板无法导入 |

## 🚫 明确不做的事

本产品专注于结果数据的安全修订，以下不在范围内：

- ❌ SQL 开发平台 / 数据库建模工具
- ❌ BI 分析与数据可视化
- ❌ 任意格式 Excel 自由导入
- ❌ 多表联动写入
- ❌ 在线协同编辑
- ❌ 业务指标计算引擎
- ❌ 主数据治理平台
