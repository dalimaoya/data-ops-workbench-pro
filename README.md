## 这是什么？

数据运维工作台是一个**轻量级的数据安全修订平台**，专为政企项目交付和运维现场设计。让普通运维和业务人员在完全不接触数据库客户端的前提下，通过「平台模板 + 校验预览 + 安全回写」的标准流程，完成结果数据表的日常维护。

它不是数据库开发工具，不是 BI 分析平台，也不是传统的数据填报系统——它只聚焦一件事：**让改数据这件事变得安全、可控、可追溯。**

### 核心亮点

- **安全回写** — 写前校验、差异预览、AI 影响评估、自动备份、一键回退，五重保障
- **平台模板** — 所有修订通过受控模板完成，杜绝任意 Excel 和手写 SQL
- **格式自适应** — 导入自动处理日期/金额/百分比/布尔格式转换，全角半角/空格/空值自动清洗
- **7 种数据库** — MySQL / PostgreSQL / SQL Server / Oracle / 达梦(DM) / 人大金仓(KingbaseES) / SQLite
- **统一认证** — 微信扫码登录 + 账号密码双模式，断网自动降级
- **四级角色** — 超级管理员 / 管理员 / 操作员 / 只读，层级管控 + 数据源级 + 插件级权限隔离
- **18 个插件** — 7 个内置 + 11 个扩展，按需启用，试用期 + 激活码授权
- **SQL 操作台** — 手写/可视化/AI 三模式只读查询
- **中英文双语** — 完整的国际化支持，一键切换
- **审批流** — 可选的回写审批机制，操作员提交 → 管理员审核 → 才执行
- **零依赖部署** — 拷贝文件夹即可运行，不需要安装 Python / Node / 数据库

## 解决什么问题？

在项目交付和运维现场，修改数据库中的结果数据是高频操作，但现有方式普遍存在风险：

| 现状痛点 | 具体表现 | 工作台方案 |
|---|---|---|
| 直接改库风险高 | 用 Navicat 写 SQL，一条 UPDATE 写错就不可逆 | 平台模板 + 校验预览 + 写前备份 |
| 线下 Excel 低效 | 导出 → 微信传 → 人工粘贴回库，版本混乱 | 平台受控模板，嵌入元信息防篡改 |
| 口头改数无留痕 | 谁改的、改了什么、改了多少，事后无从查证 | 全程操作日志 + 逐字段变更明细 |
| 格式混乱易出错 | 日期格式不统一、金额有千分位、全角半角混用 | 数据格式自适应引擎自动清洗和转换 |
| 临时脚本不可见 | 运维写临时脚本处理，过程不透明 | 标准化流程 + 版本回退 |
| 普通人员无法操作 | 每次改数据都要找研发 | 研发一次配置，用户反复使用 |

## 技术栈

| 层级 | 技术 |
|---|---|
| 前端 | React 19 + TypeScript + Ant Design 6 + Vite + Recharts |
| 后端 | Python 3.11 + FastAPI + SQLAlchemy 2.0 + Pydantic 2 |
| 认证 | JWT + bcrypt + 统一认证平台（RS256 离线验签） |
| 国际化 | react-i18next（前端）+ 后端 i18n 中间件 |
| 平台数据库 | SQLite（零配置，随文件夹走） |
| Excel 处理 | openpyxl（后端） |
| 数据处理 | format_engine（格式自适应）+ rules_engine（字段别名） |
| 打包 | PyInstaller --onedir（Windows / Linux） |
| CI/CD | GitHub Actions 自动编译发布 |

## 快速启动

### 方式一：Linux 服务器部署

```bash
chmod +x start.sh
./start.sh
```

### 方式二：Windows 桌面安装包

双击运行 `DataOpsWorkbench.exe`（pywebview 桌面窗口，内置浏览器）

### 方式三：源码运行（开发/测试）

**前置条件：** Python 3.11+ / Node.js 18+

```bash
# 后端
cd backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 9590

# 前端（开发模式）
cd frontend
npm install && npm run dev
```

### 方式四：Docker

```bash
docker-compose up -d
```

### 启动后

| 项目 | 地址 |
|---|---|
| 系统首页 | http://localhost:9590 |
| API 文档 | http://localhost:9590/docs |
| 默认账号 | `admin` / `dalimaoya`（超级管理员） |

> 首次登录后请立即修改默认密码！联网环境下优先使用微信扫码登录。

## 安全说明

| 安全措施 | 说明 |
|---|---|
| 微信扫码 | 统一认证平台 + RS256 JWT，断网缓存公钥离线验签 |
| 登录验证码 | 图形验证码防暴力破解，5 分钟有效，一次性使用 |
| JWT 认证 | 24 小时有效期，自动生成密钥 |
| 密码哈希 | bcrypt 加密存储，不存明文 |
| 权限校验 | 前端菜单隔离 + 后端 API 层角色拦截 |
| 数据源隔离 | 非授权数据源完全不可见 |
| 插件权限 | 管理员可控制用户的插件使用范围 |
| 数据源密码 | 加密存储，前端不回显 |
| 操作审计 | 所有操作记录日志，含操作人、时间、内容 |
| 写前备份 | 每次回写前自动全表备份 |
| 模板防篡改 | 嵌入隐藏元信息，非平台模板无法导入 |
| 格式校验 | 数据格式自适应引擎自动清洗和转换，防止脏数据写入 |
| 速率限制 | API 限流 200 次/分钟 |

| 目录 | 内容 | 备份建议 |

## 明确不做的事

本产品专注于结果数据的安全修订，以下不在范围内：

- SQL 开发平台 / 数据库建模工具
- BI 分析与数据可视化
- 任意格式 Excel 自由导入
- 多表联动写入
- 在线协同编辑
- 主数据治理平台
