# 数据运维工作台 (Data Ops Workbench)

> 不碰数据库客户端，也能安全修订结果表。

[![GitHub](https://img.shields.io/badge/GitHub-dalimaoya-blue?logo=github)](https://github.com/dalimaoya/data-ops-workbench)
[![Gitee](https://img.shields.io/badge/Gitee-dalimaoya-red?logo=gitee)](https://gitee.com/dalimaoya/data-ops-workbench)

---

## 这是什么？

数据运维工作台是一个**轻量级的数据安全修订平台**，让普通运维和业务人员在完全不接触数据库客户端的前提下，通过"平台模板 + 校验预览 + 安全回写"的标准流程，完成结果数据表的日常维护。

它不是数据库开发工具，不是 BI 分析平台，也不是传统的数据填报系统——它只聚焦一件事：**让改数据这件事变得安全、可控、可追溯。**

---

## 解决什么问题？

在项目交付和运维现场，修改数据库中的结果数据是高频操作，但现有方式普遍存在风险和效率问题：

| 现状痛点 | 具体表现 |
|---|---|
| 🔓 直接改库风险高 | 用 Navicat、DBeaver 等客户端直接写 SQL 改数据，一条 UPDATE 写错就可能造成不可逆后果 |
| 📎 线下 Excel 传递低效 | 导出数据 → 邮件/微信传 Excel → 人工复制粘贴回库，版本混乱、容易出错 |
| 🗣️ 口头通知改数无留痕 | 业务方口头或消息通知研发改数据，改了什么、改了多少、谁改的，事后无从查证 |
| 🔧 临时脚本不可见 | 运维写临时脚本处理数据，过程不透明，出了问题难以追溯和回退 |
| 🚫 普通人员无法自主操作 | 每次改数据都要找研发，研发变成改数据的"人肉接口" |

**数据运维工作台把这些问题一次性解决：**

- 用**平台受控模板**替代手写 SQL 和线下 Excel
- 用**自动校验 + 差异预览**替代人工比对
- 用**写前备份 + 版本回退**替代"改错了就没办法"
- 用**操作日志**替代"不知道谁改的"
- 用**研发一次配置、用户反复使用**替代"每次都找研发"

## 面向谁？

| 角色 | 典型用户 | 使用场景 |
|---|---|---|
| **配置角色** | 研发工程师、实施工程师、系统管理员 | 一次性配置数据源、纳管表、字段规则，配好之后交给使用者 |
| **使用角色** | 普通运维人员、业务数据维护人员 | 日常查看数据、下载模板、本地修改、上传校验、确认回写 |
| **管理角色** | 项目负责人、高级管理员 | 查看操作日志、在出问题时执行版本回退 |

核心理念：**研发配置一次，普通用户可以反复安全使用。**

## 核心使用场景

| 场景 | 说明 |
|---|---|
| **结果表日常修订** | 下载平台模板 → 本地用 Excel 修改 → 上传回平台 → 系统自动校验 → 预览差异 → 确认回写 |
| **专题数据临时调整** | 不用找研发写 SQL，运维人员自己就能安全完成 |
| **汇报前批量修正** | 多个指标需要批量修改，差异预览确认无误后统一写入 |
| **表结构变化后重新适配** | 管理员补充新字段定义，用户继续按原流程使用 |

## 设计思路

### 安全第一

- **写前必校验**：上传模板后系统自动校验数据类型、必填项、枚举值、字段完整性，不合格不允许写入
- **写前必预览**：逐行逐字段展示原值与新值的差异，确认后才执行写入
- **写前必备份**：每次写入前自动全表备份，保留最近 3 个版本
- **写后可回退**：出问题可一键回退到任意备份版本

### 平台模板机制

- 所有数据修订必须通过平台导出的标准模板完成
- 模板中嵌入隐藏元信息（版本、表标识等），非平台模板无法上传
- 杜绝任意来源的 Excel 直接导入

### 最小权限原则

- 三角色分权：管理员配置、操作员执行、只读用户查看
- 普通用户全程不接触数据库连接信息
- 所有操作留痕可查

### 一次配置、反复使用

- 研发/管理员完成数据源和表配置后，后续修订流程完全由普通用户独立完成
- 将研发的一次性劳动转化为团队可重复执行的标准流程

## 主链路

### 管理员配置链路（一次性）

```
新建数据源 → 测试连接 → 选择纳管表 → 自动拉取字段 → 配置字段语义与编辑规则 → 指定主键 → 保存
```

### 用户操作主链路（日常重复）

```
选择目标表 → 浏览当前数据 → 导出平台模板 → 本地 Excel 修改 → 上传模板
→ 系统校验 → 差异预览 → 确认回写 → 备份+写入 → 结果反馈 → 日志留痕
```

### 异常处理

- **校验失败** → 导出错误明细 → 本地修正 → 重新上传
- **表结构变化** → 自动检测并阻断写入 → 管理员重新确认配置
- **需要回退** → 版本回退页 → 选择版本 → 确认回退

## 功能模块

| 模块 | 核心能力 |
|---|---|
| 数据源管理 | 新增/编辑/删除/测试连接，支持 MySQL、PostgreSQL、SQL Server |
| 纳管表配置 | 选表、自动拉取字段、配置字段语义与编辑规则 |
| 表数据浏览 | 分页浏览、关键字搜索、字段筛选 |
| 模板导出 | 导出当前筛选/全量数据，含隐藏元信息 |
| 模板导入 | 上传平台模板，校验合法性/版本/字段完整性/数据类型 |
| 差异预览 | 原值/新值逐行逐字段对比，确认后才写入 |
| 安全回写 | 写前全表备份，执行写入，记录日志 |
| 表结构变化检测 | 每次进入表或导入前自动比对，不一致则阻断 |
| 日志中心 | 系统日志 / 导出日志 / 导入日志 / 回写日志 |
| 版本回退 | 备份版本列表、一键回退 |
| 登录权限 | 管理员 / 操作员 / 只读用户 三角色 |

## 明确不做的事

本产品专注于结果数据的安全修订，以下不在范围内：

- ❌ SQL 开发平台 / 数据库建模工具
- ❌ BI 分析与数据可视化
- ❌ 复杂审批流与消息通知
- ❌ 任意格式 Excel 自由导入
- ❌ 多表联动写入
- ❌ 在线协同编辑
- ❌ 业务指标计算引擎
- ❌ 主数据治理平台

| 层级 | 技术 |
|---|---|
| 前端 | React 19 + TypeScript + Ant Design 6 + Vite |

## 产品形态

**交付物是一个完整的文件夹**，拷贝到目标机器即可运行：

- Linux / macOS → 运行 `./start.sh`
- Windows → 双击 `start.bat`
- 浏览器访问 `http://localhost:8580`
- 无需安装 Python、Node、数据库等额外环境
- 支持单机部署、内网服务器部署、局域网多人访问

## 快速启动

### 前置条件

- Python 3.9+
- Node.js 18+（仅首次构建前端需要，构建完成后不再需要）

### Linux / macOS

```bash
chmod +x start.sh
./start.sh
```

### Windows

```cmd
start.bat
```

### Docker

```bash
# 使用 docker-compose
docker-compose up -d

# 或手动构建运行
docker build -t data-ops-workbench .
docker run -d -p 8580:8580 -v ./data:/app/data -v ./backups:/app/backups data-ops-workbench
```

### 启动后

- 访问地址：**http://localhost:8580**
- 默认管理员账号：`admin` / `dalimaoya`
- API 文档：**http://localhost:8580/docs**

## 独立打包

可以使用 PyInstaller 将整个项目打包为独立发布包，用户无需安装 Python 或 Node.js 即可运行。

### Linux 打包

```bash
chmod +x build.sh
./build.sh
```

### Windows 打包

> **注意：** Windows 打包必须在 Windows 环境下执行（PyInstaller 生成的可执行文件是平台相关的）。

**前置条件：**
- Python 3.9+（已加入 PATH）
- Node.js 18+（仅首次构建前端时需要；如 `backend/web/` 已有构建产物则不需要）
- pnpm 或 npm（前端包管理器）

**步骤：**
```cmd
build.bat
```

构建脚本会自动完成：
1. 创建 Python 虚拟环境并安装后端依赖
2. 安装 PyInstaller
3. 构建前端（如尚未构建）
4. PyInstaller 打包后端为独立可执行文件
5. 组装发布目录 `dist\data-ops-workbench\`

**打包产物结构：**
```
dist\data-ops-workbench\
├── start.bat          # Windows 启动脚本
├── start.sh           # Linux/macOS 启动脚本
├── server\app\        # 可执行文件及所有依赖
├── data\              # 运行时数据目录
├── backups\           # 备份存储目录
├── logs\              # 日志目录
└── README.md
```

**使用打包产物：**
```cmd
cd dist\data-ops-workbench
start.bat
```
浏览器访问 `http://localhost:8580` 即可使用。
