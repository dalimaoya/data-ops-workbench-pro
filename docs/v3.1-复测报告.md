## 复测明细

### BUG-001 [严重] PDF 对比报告生成崩溃

| 项 | 内容 |
|----|------|
| **原问题** | 导出 PDF 对比报告时触发 `NameError: SessionLocal`，返回 500 |
| **复测步骤** | 1. 创建 SQLite 数据源及纳管表 → 2. 导出当前数据 → 3. 修改 2 个字段后导入 → 4. 确认差异预览正常（diff_count=2）→ 5. 调用 `GET /api/data-maintenance/import-tasks/18/diff-report` |
| **复测结果** | ✅ **通过** — HTTP 200，成功返回 Excel 格式对比报告（5,237 bytes），不再 500 |

### BUG-002 [中等] 定时任务执行后状态误标为 failed

| 项 | 内容 |
|----|------|
| **原问题** | 定时任务立即执行后，状态显示 `failed`（`next_run_time` 属性访问异常） |
| **复测步骤** | 1. 创建 health_check 类型定时任务 → 2. 调用 `POST /api/scheduler/tasks/1/run` 立即执行 → 3. 查询执行历史 |
| **复测结果** | ✅ **通过** — 执行历史 status=`success`，result_summary 正常显示 "Health check done: 3 ok, 0 errors" |

### BUG-003 [轻微] 定时任务 next_run 字段始终为 null

| 项 | 内容 |
|----|------|
| **原问题** | 创建定时任务后 `next_run` 字段为 null / `-` |
| **复测步骤** | 1. 创建定时任务（cron: 每天 03:00）→ 2. 检查创建响应中的 next_run → 3. 执行后再次检查列表中的 next_run |
| **复测结果** | ✅ **通过** — 创建时 next_run=`2026-03-25T03:00:00`，执行后仍正确保持该值 |
