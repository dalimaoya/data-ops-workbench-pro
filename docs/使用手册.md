## 一、系统概述

数据运维工作台是一个轻量级的数据安全修订平台，帮助运维和业务人员在不直接接触数据库的前提下，通过标准化流程完成结果数据表的查看、修改、新增和删除，全程操作留痕、可追溯、可回退。

### 1.1 角色说明

| 角色 | 说明 | 权限范围 |
|---|---|---|
| **管理员 (Admin)** | 系统配置和管理负责人 | 所有功能：数据源管理、表配置、数据维护、版本回退、用户管理、审批、系统设置 |
| **操作员 (Operator)** | 日常数据维护执行人 | 数据维护（查看/编辑/导出/导入/回写/新增/删除）、日志查看、个人设置 |
| **只读用户 (Readonly)** | 数据查阅人员 | 数据查看、模板导出、日志查看、个人设置 |

### 1.2 核心流程

```
管理员一次性配置：
  新建数据源 → 测试连接 → 选择纳管表 → 配置字段 → 指定主键

用户日常操作：
  选表 → 浏览数据 → 导出模板 → Excel 修改 → 上传 → 校验 → 差异预览 → 回写
```

## 二、登录与个人设置

### 2.1 登录

1. 浏览器访问系统地址（默认 `http://localhost:8580`）
2. 输入用户名、密码和图形验证码
3. 点击登录

> 💡 验证码 5 分钟内有效，点击验证码图片可刷新。每个验证码仅可使用一次。

**默认管理员账号：** `admin` / `dalimaoya`（首次登录后请立即修改密码）

### 2.2 修改密码

1. 点击页面右上角的用户头像/用户名
2. 选择「修改密码」
3. 输入旧密码 → 输入新密码 → 确认新密码
4. 点击保存

### 2.3 修改显示名

1. 点击页面右上角的用户头像/用户名
2. 选择「个人设置」
3. 修改显示名 → 保存

## 四、管理员配置流程（一次性）

完成以下配置后，普通用户即可独立使用系统进行日常数据维护。

### 4.1 新建数据源

**路径：** 数据源管理 → 新建数据源

1. 填写基本信息：
   - **数据源名称：** 自定义名称（如"生产库-MySQL"、"报表库-PG"）
   - **数据库类型：** 下拉选择（MySQL / PostgreSQL / SQL Server / Oracle / 达梦 / 人大金仓 / SQLite）
   - **主机地址：** 数据库服务器 IP 或域名
   - **端口：** 选择数据库类型后自动填充默认端口，可手动修改
   - **用户名 / 密码：** 数据库连接凭证
   - **数据库名：** 目标数据库名称（必填）
   - **Schema：** PostgreSQL / SQL Server 需要时填写（可选）

2. 特殊类型说明：
   - **SQLite：** 只需填写数据库文件路径（如 `/data/mydb.sqlite`），无需主机/端口/用户名/密码
   - **Oracle：** 需要填写 SID 或 Service Name
   - **达梦/金仓：** 连接参数与 PostgreSQL 类似

3. 点击 **测试连接** 确认连通性
4. 点击 **保存**

### 4.2 新建纳管表

**路径：** 表配置管理 → 新建纳管表

**第一步：选择表**
1. 选择已创建的数据源
2. 系统自动拉取该数据源下的所有表
3. 使用搜索框快速找到目标表 → 点击选择

**第二步：基本配置**
1. 填写表别名（中文名称，方便识别，如"客户信息表"）
2. 配置主键字段
   - 单主键：直接输入字段名（如 `id`）
   - 复合主键：用英文逗号分隔（如 `id,code`）
3. 点击 **保存并拉取字段**

### 4.3 字段配置

保存纳管表后自动进入字段配置页面。对每个字段可配置：

| 配置项 | 说明 | 默认值 |
|---|---|---|
| **展示** | 是否在数据浏览页显示 | ☑ 开启 |
| **可编辑** | 是否允许修改 | 非主键/非系统字段默认开启 |
| **可编辑角色** | 哪些角色可编辑该字段 | 全部可编辑角色 |
| **必填** | 导入时是否必须有值 | 根据数据库 NOT NULL 自动设置 |
| **参与导出** | 是否出现在导出模板中 | ☑ 开启 |
| **参与导入** | 是否参与导入校验和回写 | ☑ 开启 |
| **别名** | 中文显示名 | 空（使用原字段名） |
| **枚举值** | 限制可填值，逗号分隔 | 空 |
| **长度限制** | 最大字符长度 | 根据数据库字段定义自动设置 |

> 💡 系统字段（如 created_at、updated_at 等）会被自动识别并标记，默认不参与导出和导入。

### 4.4 表结构检查

当数据库中的表结构发生变化时（新增/删除/修改字段），系统会自动检测并提示。

1. 进入表配置 → 点击目标表的「检查结构」
2. 系统对比当前配置与数据库实际字段
3. 如有差异：点击「同步字段」→ 重新配置新增字段的属性

## 五、日常操作流程

### 5.1 数据浏览

**路径：** 数据维护 → 选择表 → 进入维护

| 功能 | 操作 |
|---|---|
| **分页浏览** | 底部分页器，可调整每页条数 |
| **全局搜索** | 搜索框输入关键字，模糊匹配所有文本字段 |
| **字段筛选** | 点击「字段筛选」展开，选择字段和条件 |
| **数据源筛选** | 表很多时，通过数据源下拉快速缩小范围 |

### 5.2 模板导出

1. 在数据浏览页点击 **导出模板**
2. 选择导出范围：
   - **全量数据：** 导出该表所有记录
   - **当前筛选结果：** 仅导出符合筛选条件的记录
3. 确认后下载 Excel 文件
4. 大表（> 5000 行）自动走后台异步导出，完成后通知提醒

> ⚠️ 模板中的 `_meta` 工作表包含系统元信息，请勿删除或修改！

**批量导出：** 在数据维护列表页勾选多张表 → 点击「批量导出」→ 系统打包为 ZIP 下载

### 5.3 模板修改

下载的模板使用了 Excel 工作表保护：

| 区域 | 保护状态 | 说明 |
|---|---|---|
| 表头行 | 🔒 锁定 | 列名不可修改 |
| 主键列（已有数据） | 🔒 锁定 | 已有行的主键不可改 |
| 系统字段 | 不导出 | 不出现在模板中 |
| 可编辑字段数据区 | ✅ 可编辑 | 正常修改 |
| 底部空白行 | ✅ 可编辑 | 用于新增数据 |

修改完成后保存 Excel 文件。

### 5.4 模板导入与校验

1. 在数据浏览页点击 **导入模板** → 选择文件
2. 点击 **开始校验**
3. 系统自动执行多维度校验：

| 校验项 | 说明 |
|---|---|
| 模板合法性 | 是否为平台导出的模板（检查 _meta 工作表） |
| 版本一致性 | 模板版本与当前配置版本是否匹配 |
| 列数校验 | 列数是否与定义一致 |
| 列名校验 | 列头名称是否与定义一致 |
| 数据类型 | 每个字段的值是否符合数据库类型要求 |
| 必填校验 | 必填字段是否有值 |
| 枚举值校验 | 有枚举限制的字段是否在允许范围内 |
| 长度校验 | 文本长度是否在限制范围内 |
| 主键不可变 | 已有行的主键是否被修改 |

4. 校验结果查看：
   - **全部通过：** 点击「查看差异预览」
   - **有错误：** 点击「导出错误明细」→ 修正 → 重新上传
   - **快捷重试：** 在导入日志中可对失败的记录点击「重新校验」

### 5.5 差异预览与回写

1. 差异预览页面展示逐行逐字段对比：
   - 🔵 **蓝色行：** 更新（显示原值 → 新值）
   - 🟢 **绿色行：** 新增
   - 🔴 **红色行：** 删除（如支持）
2. 可点击「导出对比报告」导出 Excel 存档
3. 确认无误后点击 **确认写入**
4. 系统自动执行：
   - 全表备份（保留最近 3 个版本）
   - 逐行执行 UPDATE / INSERT / DELETE
   - 记录回写日志（含逐字段变更明细）
5. 写入结果页显示成功/失败行数

> 💡 如果开启了审批流，操作员点击「确认写入」后不会直接执行，而是提交给管理员审批。

### 5.6 在线编辑（快捷修改）

适用于只改少量数据、不想走模板流程的场景：

1. 在数据浏览页点击 **编辑模式**
2. 可编辑字段变为输入框，主键和系统字段保持只读
3. 直接修改需要更改的值（修改过的单元格黄色高亮）
4. 点击 **保存修改** → 弹出差异预览确认
5. 确认后自动备份 + 更新 + 记录日志

### 5.7 新增行

**单条新增：**
1. 在数据浏览页点击 **新增行**
2. 在弹出表单中填写各字段值
3. 保存后数据直接 INSERT 到数据库

**批量新增：**
1. 点击 **新增行** → 弹出批量新增弹窗
2. 默认显示 5 行空白行
3. 可逐行填写，或从 Excel 复制多行数据直接粘贴（Tab 分隔自动解析）
4. 需要更多行可点击「添加更多行」
5. 点击 **确认新增** → 自动过滤空行 → 批量 INSERT

### 5.8 删除行

1. 在数据浏览页勾选需要删除的行
2. 点击 **删除选中行**
3. 确认弹窗显示待删除行数
4. 确认后：自动备份 → 执行删除 → 记录日志

## 六、审批流

当管理员开启审批流后，操作员提交的回写操作需要经过管理员审核才会执行。

### 6.1 开启审批流

**路径：** 系统设置 → 审批流开关 → 开启

| 设置状态 | 效果 |
|---|---|
| **关闭（默认）** | 所有角色提交回写直接执行 |
| **开启** | 操作员提交后进入待审批状态；管理员提交仍直接执行 |

### 6.2 操作员提交审批

1. 完成差异预览后点击「确认写入」
2. 审批流开启时，系统提示："已提交审批，等待管理员审核"
3. 审批请求出现在管理员的审批中心

### 6.3 管理员审批

**路径：** 审批中心（仅管理员可见）

1. 查看待审批列表：申请人、表名、操作类型（回写/删除）、申请时间
2. 点击 **审批** → 查看差异预览详情
3. 操作：
   - **通过：** 系统自动执行回写 → 通知申请人
   - **拒绝：** 输入拒绝原因 → 通知申请人

## 九、版本回退

当回写出错或需要恢复历史数据时，管理员可执行版本回退。

**路径：** 版本回退（仅管理员可操作）

1. 使用筛选条件找到目标备份版本（按表、时间筛选）
2. 点击查看版本详情：备份时间、数据量、操作人
3. 点击 **回退到此版本**
4. 系统自动：先备份当前数据 → 恢复到选定版本
5. 回退操作本身也会记录在日志中

> 💡 系统保留每张表最近 3 个备份版本。每次回写和回退都会自动创建新备份。

## 十四、AI 智能助手

系统内置 AI 智能辅助功能，帮助提升配置效率和数据质量。

### 14.1 AI 配置

**路径：** 系统设置 → AI 智能助手（仅管理员可见）

**AI 引擎三种模式：**

| 模式 | 说明 | 适用场景 |
|---|---|---|
| 内置规则 | 纯规则引擎，无需模型 | 基础辅助，离线可用 |
| 本地模型 | 连接本地部署的大模型 | 政企内网有私有化模型 |
| 云端大模型 | 连接云端 API | 最佳体验 |

预设 9 个常见大模型平台（DeepSeek、通义千问、硅基流动、智谱、百度文心、Kimi、OpenAI、Claude、零一万物），选择后自动填充 API 地址。

本地模型和云端模型配置独立保存，切换不丢数据。

### 14.2 智能字段配置

**入口：** 纳管表 → 字段配置 → 🤖 AI 智能推荐

1. 点击「AI 智能推荐」按钮
2. AI 根据字段名、数据类型和采样数据自动推荐：
   - 中文语义名（如 `create_time` → 创建时间）
   - 是否只读（自增 ID、时间戳字段）
   - 枚举值（采样去重推荐）
3. 每条推荐有理由和置信度
4. 可逐条采纳、全部采纳、仅采纳语义名
5. 采纳后需手动点保存才生效

### 14.3 自然语言数据查询

**入口：** 数据浏览页 → 🤖 AI 查询

1. 点击「AI 查询」按钮展开查询面板
2. 用自然语言描述查询条件，如"找出最近7天更新过的记录"
3. AI 解析为结构化筛选条件并展示
4. 确认后执行查询
5. 支持追问补充条件

### 14.4 AI 批量修改

**入口：** 数据浏览页 → 🤖 AI 批量修改

1. 点击「AI 批量修改」按钮
2. 用自然语言描述修改规则，如"把部门是华北区的记录负责人改成李明"
3. AI 生成修改预览（哪些行改什么值）
4. 确认后进入差异预览 → 走正常回写流程
5. 支持单表/多表/全局生效范围

### 14.5 导入智能校验

导入模板后，系统自动执行 AI 智能校验（在原有硬规则校验之上）：
- 异常值检测：数值超出历史 P5-P95 范围
- 格式一致性：文本格式不统一
- 重复值预警：非主键列异常重复
- 跨字段逻辑：日期先后矛盾等

AI 警告以独立 Tab 展示，默认不阻断导入。

## 十五、数据库维护（批量操作）

**路径：** 数据维护 → 数据库维护

数据库维护是数据表维护的批量版，操作粒度从单表提升到库级。两个模块共享同一套纳管表配置。

### 15.1 批量纳管

1. 选择数据源 → 系统拉取该库所有表
2. 已纳管表标 ✅（可点击跳转到数据表维护）
3. 勾选未纳管的表（支持搜索/全选/反选）
4. 点击「批量纳管」→ 逐表拉取字段 + AI 智能推荐
5. 逐表确认配置（表别名、主键、字段属性）

### 15.2 批量导出

1. 勾选多张已纳管表
2. 选择导出方式：
   - ZIP 包（每表一个 xlsx）
   - 单文件（一个 xlsx 多 Sheet）

### 15.3 多表变更对比

批量操作涉及多张表时，系统提供逐表对比导航：
- 变更总览：N 张表受影响，M 条记录变更
- 逐表查看变更明细（原值/新值对比）
- 导航：上一个表 / 下一个表
- 操作：确认当前表 / 批量确认全部
- 进度条实时更新

## 十七、常见问题（FAQ）

### 登录相关

**Q：登录时验证码总是错误怎么办？**
A：点击验证码图片刷新，确保输入的是最新的验证码。验证码区分大小写。如果持续失败，清除浏览器缓存后重试。

**Q：忘记密码怎么办？**
A：联系管理员重置密码。管理员在用户管理页面可以为任何用户重置密码。

### 模板相关

**Q：模板上传失败怎么办？**
A：常见原因：
- 上传的不是平台导出的模板（缺少 _meta 工作表）→ 请从系统重新导出
- 列数或列名被修改 → 请勿修改表头行
- 模板版本与当前配置不一致（表字段有变化）→ 重新导出最新模板

**Q：导出的模板为什么有些字段不能编辑？**
A：模板使用了 Excel 工作表保护。表头行和主键列锁定防止误操作，系统字段不导出。只有可编辑字段的数据区域允许修改。

**Q：可以用自己的 Excel 直接导入吗？**
A：不可以。系统只接受平台导出的标准模板，这是为了确保数据安全。

### 数据操作相关

**Q：提示"表结构已变化"怎么处理？**
A：说明数据库中的表结构与系统记录不一致。管理员需要：
1. 进入表配置 → 点击「检查结构」
2. 确认变化 → 点击「同步字段」
3. 重新配置新增字段的属性

**Q：回写失败了数据会丢失吗？**
A：不会。系统在每次回写前都会自动全表备份。可通过版本回退恢复到回写前的状态。

**Q：在线编辑和模板导入有什么区别？**
A：在线编辑适合修改少量数据（几条到十几条），操作快捷。模板导入适合批量修改，数据量大时推荐使用。两者都有差异预览和备份保障。

**Q：删除行是真删除吗？**
A：是的，执行的是数据库 DELETE 操作。但系统会在删除前自动备份，可通过版本回退恢复。

### 审批相关

**Q：审批流开启后管理员自己的操作也要审批吗？**
A：不需要。管理员提交的操作始终直接执行，审批流仅约束操作员。

**Q：审批被拒绝后可以重新提交吗？**
A：可以。修改数据后重新走导入/编辑流程，会生成新的审批请求。

### 数据库相关

**Q：支持哪些数据库？**
A：目前支持 7 种：MySQL、PostgreSQL、SQL Server、Oracle、达梦（DM）、人大金仓（KingbaseES）、SQLite。

**Q：SQLite 数据库怎么配置？**
A：新建数据源时选择 SQLite，只需填写数据库文件路径（服务器上的绝对路径），无需主机/端口/用户名/密码。

**Q：可以连接远程数据库吗？**
A：可以。填写远程数据库的 IP 地址和端口即可。确保服务器网络可达且数据库允许远程连接。

### 多语言相关

**Q：怎么切换中英文？**
A：点击页面右上角的 🌐 语言切换按钮，选择目标语言。切换后立即生效，系统会记住你的选择。

**Q：后端返回的错误信息也会切换语言吗？**
A：系统支持后端多语言。绝大部分错误信息会随语言设置切换。

### 其他

**Q：系统可以在内网环境使用吗？**
A：可以。系统完全支持内网部署，不需要外网连接。使用独立打包版本，拷贝到内网服务器即可运行。

**Q：多人可以同时使用吗？**
A：可以。系统部署在服务器后，同一局域网内多人可通过浏览器同时访问。数据操作通过备份和日志机制保障安全。

**Q：数据存在哪里？**
A：系统自身的配置数据（数据源、表配置、用户、日志等）存储在本地 SQLite 数据库中，位于 `data/` 目录。业务数据存储在用户配置的远程数据库中，系统不迁移或复制业务数据。

**Q：如何迁移到另一台机器？**
A：将整个工作台文件夹（包括 `data/` 和 `backups/` 目录）拷贝到新机器即可。数据源连接信息可能需要根据新环境调整。v3.0 版本将提供一键备份与迁移功能。
