# 数据运维工作台 - 使用手册

---

## 一、系统概述

数据运维工作台是一个轻量级的数据安全修订平台，帮助运维和业务人员在不直接接触数据库的前提下，通过标准化流程完成结果数据表的查看、修改、新增和删除，全程操作留痕、可追溯、可回退。

### 角色说明

| 角色 | 权限范围 |
|---|---|
| **管理员** | 所有功能：数据源管理、纳管表配置、字段配置、数据维护（查看/编辑/导出/导入/回写/删除）、版本回退、用户管理、日志查看 |
| **操作员** | 数据维护（查看/编辑/导出/导入/回写/删除）、日志查看 |
| **只读用户** | 数据查看、模板导出、日志查看 |

---

## 二、管理员配置流程（一次性）

完成以下配置后，普通用户即可独立使用系统进行日常数据维护。

### 2.1 新建数据源

1. 进入 **数据源管理** → 点击 **新建数据源**
2. 填写连接信息：
   - 数据源名称（自定义，如"生产库-MySQL"）
   - 数据库类型（MySQL / PostgreSQL / SQL Server / Oracle / 达梦 / 人大金仓）
   - 主机地址、端口（选择数据库类型后自动填充默认端口）
   - 用户名、密码
   - 数据库名（必填）、Schema（PostgreSQL/SQL Server 可选）
3. 点击 **测试连接** 确认连通
4. 点击 **保存**

### 2.2 新建纳管表

1. 进入 **表配置管理** → 点击 **新建纳管表**
2. 第一步：选择数据源 → 系统自动拉取该数据源下的所有表
3. 使用搜索框快速找到目标表，点击 **选择**
4. 第二步：填写基本信息
   - 表别名（中文名称，方便用户识别）
   - 主键字段（多个主键用英文逗号分隔，如 `id` 或 `id,code`）
5. 点击 **保存并自动拉取字段**

### 2.3 字段配置

保存纳管表后自动进入字段配置页面：

- **展示**：控制该字段是否在数据浏览页面显示
- **可编辑**：控制该字段是否允许用户修改（主键和系统字段默认不可编辑）
- **必填**：控制导入时该字段是否必须有值
- **参与导出**：控制该字段是否出现在导出的模板中
- **参与导入**：控制该字段是否参与导入校验
- **别名**：字段的中文显示名
- **枚举值**：限制该字段只能填写指定值（用英文逗号分隔，如 `是,否`）
- **长度限制**：限制该字段的最大字符长度

> 提示：系统字段（如 created_at、updated_at 等）会自动标记，默认不参与导出和导入。

### 2.4 用户管理

1. 进入 **用户管理**（仅管理员可见）
2. 可以新增用户、设置角色（管理员/操作员/只读）、禁用/启用账号、重置密码

## 三、日常操作流程

### 3.1 数据浏览

1. 进入 **数据维护** → 选择目标表 → 点击 **进入维护**
2. 数据以表格形式分页展示
3. 支持全局关键字搜索和按字段筛选
4. 如果表很多，可以使用数据源下拉筛选或表名搜索框快速定位

### 3.2 模板导出

1. 在数据浏览页点击 **导出模板**
2. 选择导出范围：全量数据 / 当前筛选结果
3. 确认后自动下载 Excel 文件
4. 模板中包含隐藏的元信息（_meta 工作表），请勿删除或修改

> 注意：模板有工作表保护，表头行和主键列不可修改，仅可编辑字段的数据区域允许编辑。

### 3.3 模板修改与导入

1. 用 Excel 打开下载的模板
2. 修改可编辑字段的数据（黄底/白底单元格）
3. 如需新增数据，在模板底部空白行区域填写（包括主键）
4. 保存后回到系统，点击 **导入模板** → 选择文件 → 点击 **开始校验**
5. 系统自动校验：
   - 模板合法性（必须是平台导出的模板）
   - 列数和列名是否一致
   - 数据类型、必填项、枚举值、长度限制
   - 主键是否被修改（已有行的主键不允许变更）
6. 校验通过后查看校验结果
   - 如有错误：导出错误明细 → 修正 → 重新上传
   - 如全部通过：点击 **查看差异预览**

### 3.4 差异预览与回写

1. 差异预览页面展示每行每字段的原值与新值对比
2. 更新行显示蓝色，新增行显示绿色
3. 确认无误后点击 **确认写入**
4. 系统自动执行：
   - 全表备份（保留最近 3 个版本）
   - 逐行执行 UPDATE / INSERT
   - 记录回写日志和逐字段变更明细
5. 写入结果页显示成功/失败数量

### 3.5 在线编辑（快捷修改）

适用于只改少量数据、不想走模板流程的场景：

1. 在数据浏览页点击 **编辑模式**
2. 可编辑字段变为输入框，主键和系统字段保持只读
3. 直接修改需要更改的值，修改过的单元格会黄色高亮
4. 点击 **保存修改** → 弹出差异预览确认
5. 确认后自动备份 + 更新 + 记录日志

### 3.6 新增行

1. 在数据浏览页点击 **新增行**
2. 在弹出的表单中填写各字段值
3. 点击保存，数据直接 INSERT 到数据库

### 3.7 删除行

1. 在数据浏览页勾选需要删除的行
2. 点击 **删除选中行**
3. 弹窗确认待删除行数
4. 确认后自动备份 + 删除 + 记录日志

### 3.8 批量新增行

1. 在数据浏览页点击 **新增行**
2. 弹出批量新增弹窗，显示可编辑的表格（默认 5 行空行）
3. 可以手动逐行填写，也可以从 Excel 中复制多行数据直接粘贴（系统自动解析 Tab 分隔的数据）
4. 如需更多行，点击底部"添加更多行"按钮
5. 填写完成后点击 **确认新增**，系统自动过滤空行并批量 INSERT

## 四、审批流

当管理员开启审批流后，操作员提交的回写操作需要管理员审核后才会执行。

### 4.1 开启/关闭审批流

1. 管理员进入 **系统设置** 或通过 API 设置
2. 开启后：操作员提交回写 → 进入待审批状态 → 管理员审批通过后才执行
3. 关闭后：所有角色提交回写都直接执行（默认状态）
4. 管理员提交回写始终直接执行，不受审批流影响

### 4.2 操作员提交审批

1. 操作员完成差异预览后点击"确认写入"
2. 如果审批流已开启，系统会提示"已提交审批，等待管理员审核"
3. 审批请求会出现在管理员的审批中心

### 4.3 管理员审批

1. 进入 **审批中心**（仅管理员可见）
2. 查看待审批列表：申请人、表名、操作类型、申请时间
3. 点击 **审批** → 查看差异预览 → 确认后系统自动执行回写
4. 点击 **拒绝** → 输入拒绝原因 → 操作员的请求被驳回

---

## 五、数据源权限管理

管理员可以为每个用户分配可访问的数据源，实现数据隔离。

### 5.1 配置数据源权限

1. 进入 **用户管理**
2. 点击某个用户的"数据源权限"按钮
3. 在弹窗中勾选该用户可以访问的数据源
4. 保存后，该用户只能看到被授权的数据源和对应的纳管表

### 5.2 权限规则

- **管理员**：默认可以访问所有数据源，不受限制
- **操作员/只读用户**：只能看到被授权的数据源和表
- 未授权任何数据源的用户看不到任何数据

## 六、版本回退

当回写出错或需要恢复历史数据时：

1. 进入 **版本回退**（仅管理员可操作）
2. 使用筛选条件找到目标备份版本
3. 点击查看版本详情，确认备份时间、数据量
4. 点击 **回退到此版本**
5. 系统会先备份当前数据，再恢复到选定版本
6. 回退操作会记录在日志中

---

## 七、日志中心

日志中心分为四个页签：

| 页签 | 记录内容 |
|---|---|
| **系统操作日志** | 所有操作记录：数据源管理、表配置、导出、导入、回写、删除、回退、用户管理等 |
| **模板导出日志** | 每次导出的批次号、表名、行数、文件名、操作人、时间 |
| **模板导入日志** | 每次导入的批次号、表名、文件名、校验结果（通过/失败数）、操作人 |
| **回写日志** | 每次回写的批次号、表名、更新/新增/删除数量、备份版本号，点击可查看逐字段变更明细 |

## 八、常见问题

### Q1：模板上传失败怎么办？
**A：** 常见原因：
- 上传的不是平台导出的模板（缺少 _meta 工作表）→ 请从系统重新导出
- 列数或列名被修改 → 请勿修改表头行
- 模板版本与当前配置不一致 → 重新导出最新模板

### Q2：提示"表结构已变化"怎么处理？
**A：** 说明数据库中的表结构（字段增删改）与系统记录不一致。管理员需要：
1. 进入表配置管理 → 点击该表的 **检查结构**
2. 确认变化后点击 **同步字段**
3. 重新配置新增字段的属性

### Q3：回写失败了数据会丢失吗？
**A：** 不会。系统在每次回写前都会自动全表备份。如果回写过程中出现错误，可以通过版本回退恢复到回写前的状态。

### Q4：如何修改自己的密码？
**A：** 点击页面右上角的用户名 → 选择"修改密码" → 输入旧密码和新密码。

### Q5：导出的模板为什么有些字段不能编辑？
**A：** 模板使用了 Excel 工作表保护：
- 表头行：锁定，防止误改列名
- 主键列：已有数据行的主键锁定，防止误改关联关系
- 系统字段：不参与导出
- 可编辑字段的数据区域：正常可编辑

### Q6：支持哪些数据库？
**A：** 目前支持 6 种数据库：MySQL、PostgreSQL、SQL Server、Oracle、达梦（DM）、人大金仓（KingbaseES）。

---

### Q7：审批流开启后操作员提交回写会怎样？
**A：** 操作员提交后不会直接执行，而是进入"待审批"状态。管理员在审批中心查看并审批通过后，系统才自动执行回写。管理员自己提交的回写始终直接执行。

### Q8：如何给用户分配数据源权限？
**A：** 管理员进入用户管理 → 点击用户的"数据源权限" → 勾选可访问的数据源 → 保存。操作员/只读用户只能看到被授权的数据源。

## 九、系统说明

### 产品定位

数据运维工作台不是数据库开发工具，也不是传统数据填报平台。它聚焦于一件事：**让改数据这件事变得安全、可控、可追溯。**

### 解决的痛点

| 原有方式 | 问题 | 工作台的解决方案 |
|---|---|---|
| 直接用 Navicat/SQL 改库 | 风险高，一条 UPDATE 写错不可逆 | 平台模板 + 校验预览 + 写前备份 |
| 线下 Excel 往返传递 | 版本混乱、人工复制粘贴出错 | 平台受控模板，杜绝任意 Excel |
| 口头通知研发改数据 | 改了什么、谁改的，无从查证 | 全程操作留痕 + 逐字段变更日志 |
| 运维临时脚本处理 | 过程不透明，出问题难追溯 | 标准化流程 + 版本回退 |
| 每次改数据都找研发 | 研发变成"人肉接口" | 研发一次配置，用户反复使用 |

### 核心设计原则

- **安全第一**：写前必校验、写前必预览、写前必备份、写后可回退
- **平台模板机制**：所有修订必须通过平台模板，杜绝任意 Excel 导入
- **最小权限**：三角色分权，普通用户不接触数据库连接信息
- **一次配置反复使用**：研发配好后，普通用户可独立完成日常维护
