## 1. 产品形态（重要变更）

**交付物形态：一个完整的文件夹。**

- 用户拿到文件夹后，在 Linux 上运行 `./start.sh`，在 Windows 上双击 `start.bat`
- 自动启动本地 Web 服务
- 自动打开或提示用户打开浏览器访问 `http://localhost:端口`
- 用户通过浏览器完成所有操作
- 不需要用户安装 Python / Node / 数据库等额外环境
- 文件夹可直接拷贝部署，零安装

### 文件夹结构建议

```
data-ops-workbench/
├── start.sh                # Linux/macOS 启动脚本
├── start.bat               # Windows 启动脚本
├── server/                 # 后端服务（打包后的可执行文件或嵌入式运行时）
│   ├── app                 # Linux 可执行文件
│   ├── app.exe             # Windows 可执行文件
│   └── ...
├── web/                    # 前端静态资源（构建产物）
│   ├── index.html
│   ├── assets/
│   └── ...
├── data/                   # 平台数据库 + 配置
│   └── platform.db         # SQLite 数据库
├── backups/                # 备份文件存放
├── logs/                   # 运行日志
└── README.md               # 使用说明
```

## 2. 总体架构

```
┌─────────────────────────────────────────────┐
│  文件夹（拷贝即部署）                         │
│                                              │
│  start.sh / start.bat                        │
│       ↓                                      │
│  后端服务（单进程）                            │
│  ├── 提供 REST API                           │
│  ├── 托管前端静态资源                         │
│  ├── 连接平台 SQLite 数据库                   │
│  └── 连接外部业务数据库                       │
│       ↓                                      │
│  浏览器访问 http://localhost:port             │
└─────────────────────────────────────────────┘
```

核心设计：**后端同时托管前端静态文件**，一个进程搞定，启动即可用。

## 3. 前端

| 项目 | 选型建议 | 理由 |
|---|---|---|
| 框架 | React 18+ | 生态成熟、组件库丰富 |
| UI 库 | Ant Design 5.x | 表格/表单/弹窗场景契合度高 |
| 状态管理 | Zustand 或 React Context | 轻量，MVP 不需要 Redux |
| 路由 | React Router v6 | 标准选型 |
| Excel 处理 | SheetJS (xlsx) | 前端模板导出/导入解析 |
| 构建工具 | Vite | 构建产物放 `web/` 目录，由后端静态托管 |

---

## 4. 后端

| 项目 | 选型建议 | 理由 |
|---|---|---|
| 框架 | Python FastAPI | 快速落地、支持静态文件托管 |
| ORM | SQLAlchemy 2.0 | 支持多数据库方言 |
| 数据库驱动 | pymysql / asyncpg / pyodbc | MySQL / PostgreSQL / SQL Server |
| 密码加密 | cryptography (Fernet) | 数据源密码加密存储 |
| Excel 处理 | openpyxl | 模板生成与元信息嵌入 |
| 认证 | JWT (python-jose) | 轻量登录 |
| 静态托管 | FastAPI StaticFiles | 将 `web/` 目录挂载为静态资源 |

---

## 5. 打包与跨平台

| 项目 | 方案 | 说明 |
|---|---|---|
| Python 打包 | PyInstaller | 将 FastAPI 应用打包为单文件可执行程序 |
| Linux 产物 | `server/app` | ELF 可执行文件 |
| Windows 产物 | `server/app.exe` | PE 可执行文件 |
| 平台数据库 | SQLite | 随文件夹走，零配置 |
| 启动脚本 | `start.sh` + `start.bat` | 启动服务 + 提示浏览器地址 |

**不再使用 Tauri / Electron。** 用户只需要一个浏览器。

### 启动脚本逻辑
1. 启动 `server/app`（或 `app.exe`）
2. 等待服务就绪
3. 输出访问地址 `http://localhost:8580`
4. 尝试自动打开浏览器（可选）
