## 一、DELETE 回写完整链路（v3.10 修复重点）

### 场景：用户标记删除 → 导出 → 导入 → 差异预览 → 回写删除

| 步骤 | 用户操作 | 验收结果 |
|------|---------|---------|
| 1. 导出模板 | 用户导出模板，模板最后一列为「_操作」 | ✅ 模板正确生成操作列，带批注提示"填 DELETE 标记删除" |
| 2. 标记删除 | 用户在 _操作 列填写"DELETE"或"删除" | ✅ 支持 DELETE/delete/删除 三种标记 |
| 3. 导入校验 | 上传模板后系统识别删除标记 | ✅ 修复后正确读取 operation_col_idx，识别删除行 |
| 4. 权限校验 | 表未启用删除时标记删除 | ✅ 提示"该表未启用删除功能" |
| 5. 主键不存在 | 删除的行主键在库中不存在 | ✅ 提示"主键 xxx 在数据库中不存在" |
| 6. 差异预览 | 删除行在预览中的展示 | ✅ 红色标签 + 删除线 + 浅红行背景 + 统计卡片显示删除行数 |
| 7. 执行回写 | 确认后执行 DELETE | ✅ 先 SELECT 旧值 → 再 DELETE → 字段级变更日志记录 |
| 8. 结果页 | 回写结果展示 | ✅ 显示更新数 + 新增数 + 删除数 + 失败数 |
| 9. 混合操作 | 同一批次含更新 + 新增 + 删除 | ✅ 代码按 update → insert → delete 三段顺序执行 |

**结论：DELETE 完整链路通畅 ✅**
