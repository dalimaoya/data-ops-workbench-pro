## 一、背景

数据运维工作台是一个面向运维和业务人员的数据修订平台，支持 Windows/Linux 双平台独立部署。当前产品已实现：
- 底座功能：数据源管理、表配置、数据浏览、导出/导入/回写、日志、权限、版本回退
- 基础插件（7 个）：库表管理、SQL 操作台、定时任务、健康巡检等
- 扩展插件（12 个）：AI 智能系列、数据治理、审计报告等
- 插件中心：三层架构（底座 / 基础 / 扩展），支持热加载启用/停用
- 安装包：Nuitka 编译 + Inno Setup 安装器

**v5.0 需要接入统一架构平台**，实现用户认证、功能授权、通知推送的统一管理。

## 三、需要统一架构平台提供的能力

### 3.1 用户认证（必须）

**需求：用户可以通过微信扫码登录数据运维工作台，不需要单独注册账号。**

| 项目 | 需求 |
|------|------|
| 登录方式 | 微信扫码 → 统一架构认证 → 回调我方 |
| 协议 | OAuth 2.0 / OIDC |
| 回调地址 | `GET /api/auth/oauth/callback?code=xxx&state=xxx` |
| 返回信息 | 用户唯一 ID、昵称、头像（可选）、角色/权限等级 |
| 离线模式 | 无网络时降级为本地账号登录，两套认证并存 |
| Token 续签 | 统一架构提供 refresh token 机制 |

**我方需要的接口：**
```
POST /unified-auth/token     — code 换 token
GET  /unified-auth/userinfo  — token 获取用户信息
POST /unified-auth/refresh   — 刷新 token
```

### 3.2 功能授权 / License（必须）

**需求：根据用户的授权等级，控制可使用的功能范围。**

三种授权模式：

| 模式 | 获取方式 | 可用范围 |
|------|---------|---------|
| 离线模式 | 下载安装包直接用，无需登录 | 底座 + 基础插件（7 个） |
| 在线基础 | 微信扫码登录 | 底座 + 基础插件 + 部分扩展插件 |
| 完整授权 | 激活码（公众号签到兑换） | 全部插件 + AI 引擎 |

**我方需要的接口：**
```
GET /unified-auth/license?user_id=xxx&product=data-ops-workbench
返回：{
  "level": "basic" | "pro" | "enterprise",
  "allowed_plugins": ["plugin_ai_assistant", "plugin_data_mask", ...],
  "expires_at": "2026-06-27T00:00:00Z"
}
```

**激活码流程：**
```
用户关注公众号 → 签到获积分 → 积分兑换激活码
→ 用户在工作台"系统设置 → 授权管理"页面输入激活码
→ 我方调统一架构接口验证激活码
→ 返回授权等级和允许的插件列表
```

**我方需要的接口：**
```
POST /unified-auth/activate
Body: { "activation_code": "xxx", "user_id": "xxx", "product": "data-ops-workbench" }
返回：{ "success": true, "level": "enterprise", "expires_at": "..." }
```

### 3.3 公众号通知推送（重要）

**需求：工作台的关键操作（回写、审批、异常）可以通过公众号推送通知给用户。**

| 场景 | 说明 |
|------|------|
| 回写完成 | "您在 XX 表的回写操作已完成，更新 N 行" |
| 审批待处理 | "有一条回写审批等待您处理" |
| 巡检异常 | "数据源 XX 连接异常" |
| 系统更新 | "数据运维工作台 v5.0 已发布" |

**我方需要的接口：**
```
POST /unified-notify/send
Body: {
  "user_id": "xxx",
  "template": "writeback_complete",
  "params": { "table": "sales", "rows": 50 },
  "channel": "wechat_mp"  // 公众号
}
```

### 3.4 邮件通知（可选）

如果统一架构有邮件发送能力，也希望对接：
```
POST /unified-notify/send
Body: { "user_id": "xxx", "channel": "email", "subject": "...", "body": "..." }
```
