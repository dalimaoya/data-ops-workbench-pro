## 三、发现的问题及修复情况

### 问题 #1：语言切换按钮显示硬编码中文 — 🔧 已修复
- **严重程度**: P2（中）
- **位置**: `frontend/src/layouts/MainLayout.tsx` 第 179 行、第 437 行
- **问题描述**: 语言切换下拉菜单和按钮文字使用硬编码 `'中文'`，未走 i18n
- **修复方案**: 新增 `common.langZh` / `common.langEn` i18n key，替换硬编码字符串
- **修复状态**: ✅ 已修复并验证构建通过

### 问题 #2：数据源表单数据库类型选项硬编码中文 — 🔧 已修复
- **严重程度**: P2（中）
- **位置**: `frontend/src/pages/datasource/DatasourceForm.tsx` 第 16-17 行
- **问题描述**: "达梦 (DM)" 和 "人大金仓 (KingbaseES)" 两个选项 label 为硬编码中文
- **修复方案**: 新增 `datasource.dbTypeMySQL` ... `datasource.dbTypeKingbase` 共 6 个 i18n key，组件内动态生成选项
- **修复状态**: ✅ 已修复并验证构建通过

### 问题 #3：首页数据库类型标签硬编码中文 — 🔧 已修复
- **严重程度**: P2（中）
- **位置**: `frontend/src/pages/Home.tsx` 第 198 行
- **问题描述**: `dbTypeLabel` 函数中 `dm: '达梦', kingbase: '金仓'` 为硬编码中文
- **修复方案**: 使用 `t('datasource.dbTypeDM')` / `t('datasource.dbTypeKingbase')` 替换
- **修复状态**: ✅ 已修复并验证构建通过

### 问题 #4：后端 API 消息无多语言支持 — 📋 记录待议
- **严重程度**: P3（低）
- **位置**: `backend/app/routers/*.py`（约 118 处中文消息）
- **问题描述**: 后端所有 HTTPException detail、log_operation message 均为中文硬编码，无 Accept-Language 解析机制
- **影响评估**: 前端大部分场景用自己的 i18n 提示覆盖了后端消息，仅少数直接透传 API detail 的场景会在英文模式下显示中文错误信息
- **建议方案**: 后续迭代中引入后端 i18n 中间件，根据请求头 Accept-Language 返回对应语言消息
- **修复状态**: 📋 记录到后续迭代计划，当前不阻塞使用

### 附加修复：数据源列表筛选缺少新增数据库类型 — 🔧 已修复
- **严重程度**: P2（中）
- **位置**: `frontend/src/pages/datasource/DatasourceList.tsx` 第 27-32 行
- **问题描述**: 数据源列表的数据库类型筛选下拉只有 MySQL/PostgreSQL/SQL Server，缺少 Oracle/DM/KingbaseES
- **修复方案**: 补齐全部 6 种数据库类型选项，并使用 i18n key
- **修复状态**: ✅ 已修复并验证构建通过
