# v5.0 编译打包问题记录

## 已解决的问题

### 1. app.exe 启动后无输出静默退出
- **根因**：PyInstaller 入口文件错误——打包了 `app/main.py`（只定义 FastAPI app 不启动 uvicorn），应该打包 `app_entry.py`（初始化环境 + 启动 uvicorn）
- **修复**：workflow 入口改为 `app_entry.py`，`--name app` 保持输出名

### 2. passlib.handlers.bcrypt ModuleNotFoundError
- **根因**：PyInstaller 未自动检测到 passlib 的 handler 子模块
- **修复**：添加 `--hidden-import passlib.handlers.bcrypt` 等 + `--collect-submodules passlib`

### 3. --onefile 模式 exe 无任何输出
- **根因**：`--onefile` 解压到临时目录时可能被杀毒软件拦截，且 `--noconsole` 隐藏了所有错误
- **修复**：改用 `--onedir` 模式，去掉 `--noconsole`（调试阶段）

### 4. 首页白屏（notifications.length TypeError）
- **根因**：`MainLayout.tsx` 中 `fetchNotifications` 返回数据可能为 undefined，直接调 `.length` 崩溃
- **修复**：`Array.isArray()` 防护所有 API 响应

### 5. start.bat Unicode 字符报错
- **根因**：`────` 等 Unicode 分隔线在 cmd.exe 某些编码下被当命令执行
- **修复**：替换为 ASCII 字符 `----`

### 6. 版本号不同步
- **根因**：version.txt 写 5.0.0 但 Release tag 用 5.0.x，i18n 里还是 v4.8.0
- **修复**：CI 里 `echo $VERSION > version.txt` 自动同步，i18n 更新为 v5.0.0

### 7. web 前端文件版本不一致
- **根因**：PyInstaller `--add-data "web;web"` 把编译时的旧前端嵌入 bundle，覆盖了外部最新前端
- **修复**：去掉 `--add-data "web;web"`，从 `DATA_OPS_BASE_DIR/web/` 加载外部前端

### 8. superadmin 权限不生效
- **根因**：`get_permitted_datasource_ids` 只检查 `role == "admin"`，不识别 `superadmin`
- **修复**：改为 `role in ("admin", "superadmin")`

### 9. DataOpsWorkbench.exe 窗口不显示
- **状态**：待修复
- **可能原因**：pywebview 依赖（WebView2）在 PyInstaller 打包后缺失，或 launcher.py 的 server 路径找不到
