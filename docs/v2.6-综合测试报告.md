## 三、问题列表

### BUG-001: SQLite 版本回退失败（P1 严重）

| 属性 | 值 |
|------|-----|
| **严重程度** | P1 严重 |
| **模块** | 版本回退 |
| **影响范围** | 使用 SQLite 数据源时，版本回退功能不可用 |

**现象**: 执行版本回退时报错 `no such table: test_table_pre_rb_20260324_XXXXXX_XXXX`

**根因分析**:  
`backend/app/routers/backup_version.py` 的 `rollback_version()` 函数中，回退前自动备份的代码（第 210-224 行）仅有 `mysql`/`postgresql`/`kingbase`/`sqlserver`/`oracle`/`dm` 分支，**缺少 `sqlite` 分支**。

而 `data_maintenance.py` 的 `_create_backup_table()` 函数已经正确实现了 SQLite 分支。

**复现步骤**:
1. 创建 SQLite 数据源并纳管表
2. 导入模板并执行回写（产生备份）
3. 进入版本回退，选择备份执行回退
4. 报错 `no such table`

**修复建议**:  
在 `backup_version.py` 的回退备份代码中增加 SQLite 分支：
```python
elif ds.db_type == "sqlite":
    cur.execute(f"CREATE TABLE `{pre_rollback_backup_name}` AS SELECT * FROM {qt_source}")
```
或者重构为调用 `_create_backup_table()` 公共函数，避免代码重复。
