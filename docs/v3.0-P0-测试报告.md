## 问题清单

| # | 严重度 | 模块 | 问题描述 | 建议处理 |
|---|--------|------|----------|----------|
| 1 | ⚠️ 低 | AI 配置 | 前端保存时仅发送当前 mode 的配置字段（设计合理），但如果用户在本地模式下误改了云端字段（理论上不可见），不会发送。当前实现正确，无需修改。 | 无需修改 |
| 2 | ⚠️ 低 | 平台备份 | `_restore_overwrite` 中 settings 跳过 `***` 值合理，但恢复后 AI API Key 需要用户重新输入。前端未提示此事。 | 建议在恢复成功页面增加提示"敏感信息（API Key、数据源密码）需重新配置" |
| 3 | ⚠️ 低 | AI 校验 | `_run_outlier_checks` 中数值离群检测跳过了主键字段（正确），但对文本长度异常检测未跳过主键。zbap 的 id 字段是数字型不受影响，但字符串主键可能触发误报。 | 建议在文本长度异常检测中也增加 `if field.is_primary_key: continue` |
| 4 | ⚠️ 低 | AI 配置 | `AIConfigPage.tsx` 中 `handleSave` 发送 feature_flags 时用的是 `config?.feature_flags` 而非 form 中的值，实际行为是"保存前一次 toggle 的结果"。由于 toggle 直接修改 config state，功能上正确，但逻辑链条不够清晰。 | 可接受，不影响功能 |

## 整体结论

**🟢 v3.0-P0 四个新功能全部通过测试，可以进入发布流程。**

- **AI 智能助手设置页面**：功能完整，双套配置独立保存、掩码、连接测试、7 功能开关均正常。
- **智能字段配置助手**：规则引擎 + LLM 增强双引擎工作正常，zbap 拼音字段 AI 识别准确。
- **导入数据智能校验增强**：4 类检测（异常值/格式/重复/跨字段）均可触发，前端集成完整。
- **平台数据备份与迁移**：备份/下载/删除/上传/解析/权限控制均正常，敏感信息脱敏到位。
- **回归测试**：核心功能（登录、数据浏览、导出、仪表盘、多语言、权限）未受影响。

发现 4 个低优先级问题，均不影响核心功能，可在后续迭代中优化。
