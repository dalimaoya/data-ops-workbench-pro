## 一、架构目标

将现有的一体化代码拆分为「核心（开源）+ 插件（闭源）」两部分，实现：
1. 核心功能独立可运行，开源发布
2. 高级功能以插件形式加载，闭源分发
3. 插件可热插拔——有插件就有功能，没插件核心照常用
4. 为未来多产品共用插件框架打基础

### 3.3 后端插件加载器

```python
# plugin_loader.py（开源）

import os
import json
import importlib
from fastapi import FastAPI

PLUGINS_DIR = os.path.join(os.path.dirname(__file__), "plugins")

def load_plugins(app: FastAPI):
    """扫描 plugins/ 目录，加载所有合法插件"""
    loaded = []
    
    if not os.path.isdir(PLUGINS_DIR):
        return loaded
    
    for plugin_name in sorted(os.listdir(PLUGINS_DIR)):
        plugin_path = os.path.join(PLUGINS_DIR, plugin_name)
        manifest_path = os.path.join(plugin_path, "manifest.json")
        
        if not os.path.isfile(manifest_path):
            continue
        
        with open(manifest_path) as f:
            manifest = json.load(f)
        
        # 版本兼容性检查
        # 激活状态检查（license_key / wechat_auth）
        
        try:
            # 导入插件入口模块
            module = importlib.import_module(f"app.plugins.{plugin_name}")
            
            # 调用插件注册函数
            if hasattr(module, "register"):
                module.register(app, manifest)
                loaded.append(manifest["name"])
        except Exception as e:
            print(f"[WARN] Plugin {plugin_name} failed to load: {e}")
    
    return loaded
```

### 3.4 插件入口示例

from fastapi import FastAPI

def register(app: FastAPI, manifest: dict):

### 3.5 前端插件机制

### 3.6 后端插件 API

**GET /api/plugins/loaded** — 返回已加载的插件列表（含 manifest 信息）

**GET /api/plugins/available** — 返回所有可用插件（含未激活的）

**POST /api/plugins/{name}/activate** — 激活插件（输入 license key）

## 四、激活机制

### 4.1 License Key 方式

```
用户购买 → 获取 license key → 在系统设置中输入 → 解锁对应插件
```

License key 包含：
- 插件名列表
- 有效期
- 机器码绑定（可选）
- 签名（防篡改）

### 4.2 微信公众号方式（未来）

```
用户扫码关注公众号 → 获取 openid → 后端验证订阅状态 → 解锁插件
```

## 五、分拆实施步骤

### Step 1：创建插件框架（不动现有功能）
- 新建 plugin_loader.py
- 新建 plugins/ 目录
- main.py 增加插件加载逻辑
- 前端增加插件菜单动态加载
- **此步骤完成后，现有功能不受影响（插件目录为空时等同于全功能版）**

### Step 2：逐个模块迁移为插件
按以下顺序迁移（每次迁移一个，测试通过再迁下一个）：
1. plugin-notification（最简单，先练手）
2. plugin-approval（独立性强）
3. plugin-backup（独立性强）
4. plugin-report（独立性强）
5. plugin-scheduler（独立性强）
6. plugin-health-check
7. plugin-batch-ops
8. plugin-smart-import
9. plugin-ai-assistant（最复杂，最后迁）

### Step 3：验证
- 删掉所有插件 → 核心功能正常运行
- 放回所有插件 → 完整功能正常
- 只放部分插件 → 对应功能可用，其他灰色

### Step 4：分仓库
- 核心代码留在公开仓库
- 插件代码移到私有仓库
- CI/CD 分别构建
