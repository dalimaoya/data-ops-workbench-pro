## 问题清单

| # | 严重度 | 问题描述 | 影响 |
|---|--------|----------|------|
| 1 | **高** | SQLite 数据源所有写操作返回 "attempt to write a readonly database" | 所有 SQLite 数据源（ds_id=22 和 ds_id=24）的 DDL 执行均失败。SQL 预览正常，但 execute=true 时报错。影响库表管理建表/索引/加列 + 数据维护在线编辑。 |

> **注**: 此问题在 v5.0 全平台测试中已发现（数据维护在线编辑同一报错），属于同一根因 -- SQLite 连接以只读模式打开或文件权限不足。

## 补充说明

### API 路由差异
实际 API 路由与任务描述略有不同：
- 实际路由不含路径参数 ds_id，而是通过 body/query 的 `datasource_id` 字段传递
- 所有 DDL 操作支持 `execute` 标志：false 时仅预览 SQL，true 时执行
- 删除表需要 `confirm=true` 确认，且自动创建备份表

### execute 预览/执行机制
库表管理的预览机制设计合理：默认 execute=false 仅返回 SQL，前端可展示给用户确认后再发 execute=true 执行。
