## 1. 产品一句话定义

为交付和运维现场提供一个"不碰数据库客户端也能安全修订结果表"的可视化工作台——用平台受控模板替代手写 SQL、口头改数和线下 Excel 传递，做到每一次修改都可校验、可预览、可回写、可回退、可留痕。

---

## 2. 产品定位

数据运维工作台不是数据库开发工具，也不是传统数据填报平台。

它的定位是：**围绕数据库中的结果数据表，提供"读取、模板导出、Excel 修订、校验预览、安全回写、留痕回退"的轻量工作台。**

它要替代的低效且高风险方式：
- 直接用 Navicat / SQL 改库
- 线下 Excel 往返传递，人工复制粘贴回库
- 口头通知研发改数
- 运维靠脚本临时处理，过程不可见

---

## 3. 产品目标

### 3.1 核心目标
让普通运维人员或业务维护人员，在**不直接接触数据库**的前提下，完成结果数据的安全更新。

### 3.2 次级目标
把研发的一次性配置，转换成后续普通用户可重复执行的标准流程。

### 3.3 沉淀目标
形成一个可复用、可演示、可开源外壳化的工作台成果物，用于：
- 项目交付提效
- 运维流程规范化
- 个人方法论沉淀
- 案例展示与品牌输出

---

## 4. 目标用户

| 角色 | 代表人 | 职责 |
|---|---|---|
| 配置角色 | 研发/实施工程师/管理员 | 配置数据源、表与字段语义、编辑规则、主键 |
| 使用角色 | 普通运维/业务维护人员 | 查看数据、下载模板、修改上传、确认回写 |
| 管理角色 | 项目负责人/高级管理员 | 查看日志、执行回退、管理配置状态 |

---

## 5. 核心使用场景

| 场景 | 描述 |
|---|---|
| 结果表日常修订 | 下载模板 → 修改 → 上传 → 校验 → 回写 |
| 专题数据临时调整 | 无需找研发写 SQL，运维自主完成 |
| 领导汇报前批量修正 | 批量修改多指标 → 差异预览 → 统一写入 |
| 结构变化后重新配置 | 管理员补充新字段定义，用户继续按原流程使用 |

---

## 6. MVP 功能范围

### ✅ 首版必须做

| # | 模块 | 核心能力 |
|---|---|---|
| 1 | 数据源管理 | 新增/编辑/删除/测试连接，支持 MySQL、PostgreSQL、SQL Server |
| 2 | 纳管表配置 | 选表、记录库/schema/表名、自动拉取字段、配置字段语义与编辑规则 |
| 3 | 字段配置 | 展示/可编辑/必填/主键/系统字段/参与导出导入/枚举值/长度限制 |
| 4 | 表数据浏览 | 分页、文本统一展示、关键字与字段筛选 |
| 5 | 模板导出 | 导出当前筛选/全量数据，含隐藏元信息，非平台模板禁止回传 |
| 6 | 模板导入 | 上传平台模板，校验合法性/版本/字段完整性/数据类型 |
| 7 | 导入校验 | 展示校验结果（通过/失败/警告），支持导出错误明细 |
| 8 | 差异预览 | 原值/新值逐行逐字段对比，确认后才写入 |
| 9 | 安全回写 | 写前全表备份，执行写入，记录日志，保留最近 3 个备份版本 |
| 10 | 表结构变化检测 | 每次进入表或导入前自动比对，不一致则阻断 |
| 11 | 日志中心 | 系统操作日志 / 导出日志 / 导入日志 / 回写日志 |
| 12 | 版本回退 | 备份版本列表、一键回退指定版本 |
| 13 | 基础登录 + 权限 | 管理员 / 操作员 / 只读，三角色 |

### ❌ 首版不做

- SQL 开发平台 / BI 分析 / 数据建模
- 审批流 / 消息通知
- AI 智能字段识别 / 自动映射
- 多表联动写入
- 自定义表单设计器
- 自由格式 Excel 上传
- 指标定义与计算规则建模
- 在线协同编辑

### ⏳ 明确后置

| 能力 | 后置理由 |
|---|---|
| Oracle / 达梦 / 人大金仓 | 先跑通主流三库 |
| 审批流 | 首版无多级审核需求 |
| 逐字段变更明细日志 | 批次级日志已足够 MVP |
| 复杂权限粒度 | 三角色够用 |
| 多租户 | 当前单系统运行 |
| 大表异步导出 | 首版数据量可控 |

## 7. 主链路定义

### 管理员配置链路（一次性）
```
新建数据源 → 测试连接 → 选择纳管表 → 自动拉取字段 → 配置字段语义与编辑规则 → 指定主键 → 保存配置
```

### 用户操作主链路（日常重复）
```
选择目标表 → 浏览当前数据 → 导出平台模板 → 本地修改模板 → 上传模板 → 系统校验 → 差异预览 → 确认回写 → 备份+写入 → 结果反馈 → 日志留痕
```

### 异常分支
- **校验失败** → 导出错误明细 → 本地修正 → 重新上传
- **表结构变化** → 阻断写入 → 管理员重新确认配置
- **需要回退** → 版本回退页 → 选择版本 → 确认回退

## 8. 产品形态

| 项目 | 决策 |
|---|---|
| 交付物形态 | **一个完整的文件夹**，拷贝到目标机器即可运行 |
| 运行方式 | Linux 运行 `./start.sh`，Windows 双击 `start.bat`，自动启动本地 Web 服务 |
| 用户交互 | 浏览器访问 `http://localhost:端口`，网页形式交互 |
| 跨平台 | Linux + Windows 均可运行 |
| 部署方式 | 单机拷贝部署 / 内网服务器部署 / 局域网访问 |
| 不需要 | 用户无需安装 Python / Node / 数据库等额外环境 |
| 不建议形态 | 纯静态 HTML / 纯命令行 / 纯对话式黑箱 |

---

## 9. 数据策略

| 策略 | 说明 |
|---|---|
| 展示 | 页面统一按文本形式展示字段值 |
| 模板 | 统一按文本形式导出 |
| 写入 | 系统按数据库字段类型做强制转换，转换失败则报错不写入 |
| 写入范围 | 首版仅支持更新已有记录，不支持自由新增/删除 |
| 备份 | 写入前全表备份，最多保留最近 3 个版本 |

---

## 10. 权限策略

| 角色 | 能力 |
|---|---|
| 管理员 | 管理数据源、表配置、查看全部日志、执行回退 |
| 操作员 | 查看数据、下载模板、上传模板、预览差异、提交写入 |
| 只读用户 | 查看数据、下载模板，不允许上传写入 |

---

## 11. 成功标准

- 普通用户无需接触数据库客户端即可更新结果表
- 用户必须通过平台模板完成修订
- 平台可识别错误模板并阻止写入
- 每次写入前均有备份
- 出错后可回退到最近版本
- 研发只需首配一次，后续普通用户可独立使用

---

## 12. 非目标声明

本产品不做：数据库建模工具、SQL 开发平台、BI 分析展示平台、复杂审批流平台、任意 Excel 自由导入系统、业务指标计算引擎、主数据治理平台。
