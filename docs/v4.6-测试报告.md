### 二、后端接口

#### 5. ✅ GET /api/plugins/all 返回全部插件，含 layer/category/enabled 状态

`routers/plugins.py` 的 `list_all_plugins` 端点调用 `get_all_plugins_full()`，该函数遍历 `_all_manifests`（全量扫描结果），对每个插件返回 `name, display_name, display_name_en, version, description, author, license, layer, category, enabled, loaded, frontend` 完整字段。builtin 插件 `enabled` 固定为 `True`，extension 插件根据 `plugin_status` 表查询。

#### 6. ✅ PUT /api/plugins/{id}/toggle 可启用/停用扩展插件

`toggle_plugin()` 函数正确实现了启用/停用逻辑：
- 校验插件存在性
- 校验 `layer == "extension"` 才允许操作
- 操作 `plugin_status` 表，upsert 模式
- 返回新的启用状态

#### 7. ✅ toggle 接口仅 admin 可调用（operator/viewer 返回 403）

`toggle_plugin_status` 端点使用 `Depends(require_role("admin"))`。`require_role` 函数在 `utils/auth.py` L145 定义，非指定角色返回 HTTP 403。验证通过。

#### 8. ✅ builtin 插件不允许被停用

`toggle_plugin()` 函数中 L158-159：
```python
if manifest.get("layer") != "extension":
    raise ValueError(f"Plugin '{plugin_id}' is builtin, cannot toggle")
```
返回 HTTP 400。验证通过。

### 三、插件中心页面

#### 9. ✅ 页面可正常加载，卡片展示 12 个扩展插件

`PluginCenterPage.tsx` 在 `loadPlugins` 中调用 `fetchAllPlugins()` 后过滤 `p.layer === 'extension'`，仅展示扩展插件。使用 `Row/Col` + `Card` 响应式网格布局，每个卡片含图标、名称、描述、分类标签、版本号、启停开关。

#### 10. ✅ 分类筛选（6 个标签）工作正常

`ALL_CATEGORIES` 定义了 7 个选项（含 `all`），对应 6 个分类：`ai, governance, report, workflow, integration, market`。`filteredPlugins` 的 `useMemo` 正确实现分类过滤逻辑。

#### 11. ✅ 搜索功能正常

搜索逻辑在 `filteredPlugins` 中实现，匹配 `display_name, display_name_en, description, name` 四个字段，不区分大小写。支持 `allowClear`。

#### 12. ✅ admin 可看到 Switch，可切换启用/停用

所有用户都能看到 `Switch` 组件，但非 admin 的 `Switch` 设置为 `disabled={!isAdmin}`。admin 可正常操作，调用 `togglePlugin` API 并更新本地状态。

#### 13. ⚠️ 非 admin 用户看不到 Switch 或无法操作

**现象**: 非 admin 用户仍可看到 `Switch`（只是置灰不可操作），而非完全隐藏。虽有 Tooltip 提示"仅管理员可操作"，且前后端都有权限校验，但 UX 上让普通用户看到置灰的启停开关可能引起困惑。

**严重级别**: 低 (Low) — 功能正确，属 UX 优化项
**建议**: 可接受当前实现。如需优化，可对非 admin 隐藏 Switch 或替换为状态标签。

### 四、导航栏分组

#### 14. ✅ 导航栏分四组：底座 / 基础工具 / 扩展功能 / 系统

`buildMenuGroups()` 函数构建 4 个分组：
1. **底座** (`menu.baseGroup`) — 仪表盘、数据源、表配置、数据维护
2. **基础工具** (`menu.toolGroup`) — 定时任务、健康巡检、备份迁移
3. **扩展功能** (`menu.extensionGroup`) — 动态，仅已启用扩展插件
4. **系统** (`menu.systemGroup`) — 日志中心、版本回退、用户管理、关于、插件中心

#### 15. ✅ 分组有分割线和标题

渲染代码在 `menuGroups.map` 中，`groupIdx > 0` 时渲染分割线（`height: 1, background: rgba(255,255,255,0.08)`）和分组标题（小号字、大写字母间距）。第一组不显示分割线。

#### 16. ✅ 启用扩展插件后导航栏出现对应菜单

导航栏通过 `/api/plugins/loaded` 获取插件状态列表，`buildMenuGroups` 中 `extensionPluginMenuDefs` 的遍历逻辑：`if (loadedSet.has(def.pluginName))` 将已加载的扩展插件菜单加入"扩展功能"组。

#### 17. ✅ 停用后导航栏移除

同上逻辑，未加载（`loaded: false`）的插件不会进入 `loadedSet`，因此不会出现在导航栏中。

#### 18. ✅ 无启用扩展插件时"扩展功能"组不显示

`buildMenuGroups` 中：
```js
if (extItems.length > 0) {
  groups.push({ ... });
}
```
当无已启用扩展插件时，`extItems` 为空，"扩展功能"组不会被添加到 `groups` 数组中。

#### (附加) ✅ 插件中心入口在"系统"组

`systemMenuItems` 包含 `{ key: '/plugin-center', icon: <AppstoreOutlined />, labelKey: 'menu.pluginCenter', roles: ['admin'] }`。

## 缺陷清单

### ❌ BUG-4.6-001: plugin-smart-import 分层归属不一致

| 项目 | 内容 |
|------|------|
| **现象** | `plugin-smart-import` 在 `manifest.json` 中标记为 `"layer": "extension"`，但在 `MainLayout.tsx` 的 `builtinPluginMenuDefs` 中被当作 builtin 工具处理（放入"数据维护"子菜单）。导致该插件在插件中心显示为可启停的扩展插件，但其导航菜单项却永远按 builtin 逻辑显示（只要后端加载就出现在底座组）。 |
| **严重级别** | **中 (Medium)** — 导航行为与插件中心启停状态可能不一致 |
| **影响** | 1. 用户在插件中心停用"智能数据导入中心"后，导航栏可能仍显示其入口（因为 builtin 菜单逻辑不检查 extension 状态）<br>2. 反之，如果后端未加载该插件（extension 未启用），底座组内也不会出现该菜单，但用户在插件中心启用后可能期望它出现在"扩展功能"组而非"底座"组 |
| **建议** | 方案 A：将 `plugin-smart-import` 的 `manifest.json` 改为 `"layer": "builtin"`（如果它应该始终可用）<br>方案 B：将其从 `builtinPluginMenuDefs` 移到 `extensionPluginMenuDefs`（如果它应该可启停）<br>需产品确认归属 |

## 风险/建议项

### ⚠️ RISK-4.6-001: 启停后需重启后端才生效

| 项目 | 内容 |
|------|------|
| **现象** | `toggle_plugin()` 只修改了 `plugin_status` 数据库记录，但插件的实际加载（路由注册、`register()` 调用）发生在 `load_plugins()` 启动时。启停操作后返回的 `message` 提示"刷新页面后生效"，但实际需要**重启后端服务**才能让路由生效。前端导航栏通过 `/api/plugins/loaded` 获取状态，也只反映启动时的加载状态。 |
| **严重级别** | 低 (Low) — 属已知限制，提示信息略有误导 |
| **建议** | 1. 将提示文案改为"重启服务后生效"更准确<br>2. 长期可考虑热加载机制 |

### ⚠️ RISK-4.6-002: 非 admin 用户可见 Switch（置灰状态）

见测试要点 13 的说明。功能正确，属 UX 优化项，不阻塞发布。

## 代码质量观察

1. **✅ 接口安全**: GET /all 需登录（`get_current_user`），PUT /toggle 需 admin（`require_role("admin")`），权限分层清晰。
2. **✅ 数据一致性**: `get_all_plugins_full()` 每次实时查询 `plugin_status` 表，避免缓存不一致。
3. **✅ 错误处理**: toggle 操作的 ValueError 映射为 HTTP 400，前端 catch 并显示错误信息。
4. **✅ 国际化**: 插件中心页面支持中英文切换，导航分组标题使用 i18n key。
5. **✅ 响应式布局**: 插件卡片使用 `Col xs={24} sm={12} md={8} lg={6}` 适配不同屏幕。
