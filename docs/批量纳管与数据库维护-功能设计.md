## 一、功能定位

### 1.1 解决什么问题

当前工作台的操作粒度是"单表"——一次纳管一张表、一次导出一张表、一次回写一张表。政企现场一个库动辄几十上百张表，逐张操作效率太低。

本功能把操作粒度从"单表"提升到"库级批量"：
- 批量纳管：一次选多张表，批量拉取字段 + 批量 AI 配置
- 批量维护：一次导出/导入/回写多张表

### 1.2 菜单结构调整

```
📊 数据维护（原有入口不变）
├── 📋 数据表维护（原有功能重命名，所有现有功能不动）
└── 🗄️ 数据库维护（新增）
    ├── 批量纳管 + AI 配置
    ├── 批量导出
    └── 批量导入/回写
```

## 二、批量纳管 + AI 配置

### 2.1 使用流程

```
管理员进入「数据库维护」
  │
  ├─ Step 1: 选择数据源
  │   下拉选择已配置的数据源
  │
  ├─ Step 2: 选择要纳管的表
  │   系统拉取该库所有表，展示列表：
  │   - 已纳管的表标 ✅，不可重复纳管
  │   - 未纳管的表可勾选
  │   - 支持全选/反选/搜索过滤
  │   - 显示每张表的字段数和预估行数
  │
  ├─ Step 3: 批量纳管
  │   点击「批量纳管」
  │   - 逐表自动拉取字段
  │   - 逐表调用 AI 智能字段推荐（规则引擎 + LLM）
  │   - 进度条显示：正在处理第 X/N 张表
  │
  ├─ Step 4: 批量配置确认
  │   展示所有表的 AI 推荐结果，逐表确认：
  │   - 当前表：表别名 + 字段列表（语义名、只读、枚举值等）
  │   - 管理员可手动修改任意字段
  │   - 导航：[上一个表] [下一个表]
  │   - 操作：[确认当前表] [批量确认全部]
  │   - 进度：已确认 X/N 张表
  │
  └─ Step 5: 批量保存
      所有表确认后，一键保存所有纳管表和字段配置
```

### 2.3 后端 API

**POST /api/batch-manage/tables** — 批量纳管
```json
{
  "datasource_id": 4,
  "table_names": ["ioc_szjj_hgjj_czjr", "ioc_szjj_hgjj_gdzctz", ...],
  "auto_ai_suggest": true
}
```

返回：逐表的纳管结果 + AI 字段推荐

**POST /api/batch-manage/confirm** — 批量保存配置
```json
{
  "tables": [
    {
      "table_name": "ioc_szjj_hgjj_czjr",
      "display_name": "财政金融指标",
      "primary_key": "id",
      "fields": [...]
    },
    ...
  ]
}
```

## 三、数据库批量维护

### 3.1 批量导出

```
选择数据源 → 展示已纳管的表（可全选/勾选）
  │
  ├─ 导出方式选择：
  │   ○ ZIP包（每表一个 xlsx 文件）
  │   ○ 单文件（一个 xlsx，每表一个 Sheet）
  │
  └─ [批量导出] → 后台生成 → 完成后下载
```

**后端 API：**

**POST /api/batch-maintain/export**
```json
{
  "datasource_id": 4,
  "table_ids": [3, 4, 16],
  "format": "zip"  // "zip" 或 "multi_sheet"
}
```

### 3.2 批量导入与回写

```
上传 ZIP 包（或多 Sheet xlsx）
  │
  ├─ 系统自动识别每个文件/Sheet 对应哪张纳管表
  │   （基于文件名/Sheet名 + 元信息匹配）
  │
  ├─ 逐表校验（硬规则 + AI 智能校验）
  │
  ├─ 进入多表变更对比导航（复用已开发的组件）
  │   逐表查看差异 → 逐表确认 → 批量确认
  │
  └─ 批量回写（逐表独立备份+写入+日志）
```

**后端 API：**

**POST /api/batch-maintain/import** — 上传批量文件

**POST /api/batch-maintain/validate** — 批量校验

**POST /api/batch-maintain/writeback** — 批量回写

## 四、工作量估算

| 功能 | 预估工作量 | 依赖 |
|---|---|---|
| 菜单结构调整（数据表维护/数据库维护） | 1 天 | 无 |
| 批量纳管 + AI 配置 | 4 天 | AI 字段推荐（已完成） |
| 批量导出（ZIP + 多 Sheet） | 2 天 | 无 |
| 批量导入 + 匹配识别 | 3 天 | 无 |
| 批量回写（对接多表对比） | 2 天 | 多表变更对比（开发中） |
| 批量状态总览 | 1 天 | 无 |

**总计：约 13 天**

## 五、验收标准

- [ ] 选择数据源后能看到所有表，已纳管的标 ✅
- [ ] 可批量选择多张表一键纳管
- [ ] 纳管过程中自动调用 AI 推荐，有进度条
- [ ] 逐表配置确认，支持手动修改
- [ ] 批量保存配置
- [ ] 批量导出为 ZIP 或多 Sheet xlsx
- [ ] 批量导入能自动识别对应表
- [ ] 批量回写走完整安全链路（备份+校验+预览+写入+日志）
- [ ] 菜单层级：数据表维护 / 数据库维护 分开
