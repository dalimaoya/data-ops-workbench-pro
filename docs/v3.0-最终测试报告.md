### 4. 字段敏感等级配置

| 测试点 | 结果 | 说明 |
|--------|------|------|
| 字段敏感等级 UI | ✅ | `FieldConfigPage.tsx` 中有 sensitivity_level 列和编辑下拉（普通/敏感/高敏感） |
| 读取敏感等级 | ✅ | `GET /api/ai/field-sensitivity/{table_id}` 返回各字段的 sensitivity_level |
| 设置敏感等级 | ✅ | `PUT /api/ai/field-sensitivity` 更新字段敏感等级成功 |
| 影响评估联动 | ✅ | 设置 rq 字段为"高敏感"后，修改该字段触发高风险评估（level=high, type=sensitive_field） |
| 完整链路验证 | ✅ | 配置高敏感 → 保存 → 影响评估正确识别 → 已验证 |

**发现问题 BUG-P2-001：** `FieldConfigOut` Schema 未返回 `sensitivity_level` 和 `sensitivity_note` 字段。通用字段配置 API（`GET /api/field-config/{table_id}`）的返回值缺少这两个字段。**严重级别：低**（有独立的 `/api/ai/field-sensitivity` API 可正常使用，不影响核心功能）。

**结论：✅ 通过（有 1 个低优先级 bug）**
