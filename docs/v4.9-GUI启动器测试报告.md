## 二、代码质量发现

### 🟡 建议改进项（非阻塞）

| # | 严重级别 | 现象 | 建议处理方式 |
|---|---|---|---|
| B1 | 低 | `_wait_for_ready` 超时后静默放过，用户不知道服务没起来 | 建议超时后将状态置为 `stopped` 并提示用户 |
| B2 | 低 | `_kill_port_occupier` 仅限 `win32`，用 `netstat + taskkill`；如果端口被非 Windows 系统占用则无处理 | 当前仅 Windows 场景，可接受；如后续跨平台需补充 |
| B3 | 低 | `os._exit(0)` 在 `_quit_app` 中强制退出，跳过 Python 清理 | 对桌面应用可接受，但若后续需 atexit 回调则需改为 `sys.exit` |
| B4 | 信息 | `_check_update` 请求 GitHub API，中国大陆网络可能超时 | 已有 `timeout=5` + `try/except`，降级无感，可接受 |
| B5 | 信息 | 健康检查打开 `/loading` 而非 `/`，需确认前端有该路由 | 确认前端是否有 loading 路由，否则改为 `/` |

### ✅ 代码亮点

- **可选依赖处理好**：psutil、pystray 均 try/except 降级，不影响核心功能
- **线程安全**：UI 更新都通过 `self.root.after(0, ...)` 回主线程
- **版本读取链路完整**：version.txt → package.json → 硬编码 fallback
- **端口冲突处理**：启动前主动杀占用端口的进程
- **进程退出检测**：`_wait_for_ready` 中检查 `self.process.poll()` 避免空等
- **安装器兼容**：exe 优先 + bat 兜底双路径，开机自启 Registry 也做了
