### 4. 密码存储 ⚠️ 部分通过

**4.1 bcrypt 迁移机制 ✅**

代码审计确认（`backend/app/utils/auth.py`）：
- 使用 `passlib.CryptContext(schemes=["bcrypt"])` 进行密码哈希
- `verify_password()` 同时支持 bcrypt 和 SHA256 旧格式
- `needs_password_migration()` 检查是否需要迁移
- 登录成功时自动迁移旧哈希为 bcrypt

**4.2 admin 密码已迁移 ✅**

数据库查询结果：
```
admin: $2b$12$3zRwLXCDXfOEvFze/M4XGe... （bcrypt 格式）
```

**4.3 其他账号仍为 SHA256 ⚠️**

```
operator: SHA256 (64字符十六进制)
readonly: SHA256
operator1, readonly1, readonly_test, testop, testop2, test_op, test_ro: 均为 SHA256
```

这是 **惰性迁移** 策略的预期行为——用户下次登录时自动迁移。不算缺陷，但需注意。

**4.4 创建用户密码强度校验 ❌ 缺失**

- `PUT /api/me/password`（修改密码）：**有** 密码强度校验（8位+大小写+数字）✅
- `POST /api/users`（创建用户）：**无** 密码强度校验 ❌

**代码证据：**
- `users.py` 第 94-113 行：`create_user()` 直接调用 `hash_password(body.password)` 无任何校验
- `UserCreate` schema 中 `password: str` 无 min_length 约束

**风险等级：** 中。管理员可创建弱密码用户。

**建议：** 提取密码强度校验为公共函数，在创建用户和修改密码时统一调用。

### 6. API 限流 ⚠️ 部分通过

**6.1 代码实现 ✅**

`security_middleware.py` 中的 `RateLimiter` 实现完整：
- login: 5次/分钟/IP
- captcha: 10次/分钟/IP
- export: 10次/分钟/用户
- writeback: 5次/分钟/用户
- general: 60次/分钟/用户

`auth.py` 中 captcha 和 login 接口均调用了 `check_rate_limit()`。

**6.2 localhost 豁免 ⚠️**

```python
# security_middleware.py 第 115-116 行
if identifier in ("127.0.0.1", "localhost", "::1", "testclient"):
    return True
```

**localhost/127.0.0.1 请求完全绕过限流。** 这在开发阶段合理，但如果生产环境也通过 localhost 反向代理访问，则限流失效。

**风险等级：** 低-中。取决于部署架构。

**建议：**
1. 生产环境移除 localhost 豁免，或通过环境变量控制
2. 反向代理场景下，使用 `X-Forwarded-For` 获取真实客户端 IP

### 7. 登录锁定 ✅ 通过

**代码审计确认**（`security_middleware.py` → `LoginLockout` 类）：
- 5 次失败后锁定账号
- 锁定时长 15 分钟
- 成功登录后重置计数
- 提供管理员手动解锁接口

**auth.py 中的调用链：**
1. 登录前检查 `login_lockout.is_locked(req.username)` → 已锁定返回 403
2. 密码错误时 `login_lockout.record_failure(req.username)` → 记录失败
3. 登录成功时 `login_lockout.reset(req.username)` → 重置

**注意：** 因验证码机制阻断，无法通过 API 黑盒验证锁定效果。代码逻辑审计无误。

### 10. 数据源密码 ✅ 通过

**API 层面：**
- `DatasourceOut` 响应模型（`schemas/datasource.py`）**不包含** `password` 或 `password_encrypted` 字段
- 列表和详情 API 使用 `response_model=DatasourceOut`，Pydantic 自动过滤未声明字段

**存储层面：**
- 密码存入 `password_encrypted` 字段，使用 Fernet 对称加密（`utils/crypto.py`）
- 密钥存储在 `data/.fernet_key`，首次运行自动生成
- 仅在连接测试等需要明文密码的内部逻辑中调用 `decrypt_password()`
