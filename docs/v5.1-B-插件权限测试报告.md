**根因分析**: `backend/app/routers/plugins.py:45-49` 中，authorized 字段仅根据 `has_active_trial` (试用期) 设置，未结合用户的 plugin-permissions 进行过滤：

```python
for p in plugins:
    if p.get("layer") == "builtin":
        p["authorized"] = True
    else:
        p["authorized"] = has_active_trial  # BUG: 未检查用户级别权限
```

应增加逻辑：对于非 superadmin/admin 用户，extension 插件的 authorized 应同时满足 `has_active_trial` **且** 插件在用户的 plugin_permissions 列表中。

**结果: FAIL**

## 问题清单

| # | 严重度 | 问题描述 | 影响 | 定位 |
|---|--------|----------|------|------|
| 1 | **高** | `/api/plugins/loaded` 的 authorized 字段未结合用户插件权限过滤 | operator/viewer 用户即使未被授权也能看到所有 extension 插件为 authorized=True，权限配置形同虚设 | `backend/app/routers/plugins.py:45-49` |

## 修复建议

在 `GET /api/plugins/loaded` 接口中，对于非 superadmin/admin 角色的用户：
1. 查询该用户的 plugin_permissions
2. extension 插件的 authorized = `has_active_trial AND plugin_name IN user_permissions`
3. builtin 插件保持 authorized=True
4. superadmin/admin 保持所有 authorized=True（不受限制）
