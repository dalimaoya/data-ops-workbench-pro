## 六、缺陷清单

### BUG-001 [严重] PDF 对比报告生成崩溃

- **现象**: 导出 PDF 报告时返回 500，`NameError: name 'SessionLocal' is not defined`
- **位置**: `backend/app/routers/data_maintenance.py` 第 3439 行
- **原因**: `_generate_pdf_report()` 函数使用了 `SessionLocal()` 但未从 `app.database` import
- **修复建议**: 在函数开头 `from app.database import SessionLocal`，或使用调用方传入的 `db` 参数替代
- **严重级别**: 🔴 严重（功能不可用）

### BUG-002 [中等] 定时任务执行后状态误标为 failed

- **现象**: 手动执行健康巡检，实际成功 (2 ok, 0 errors)，但执行日志状态为 `failed`
- **位置**: `backend/app/scheduler/engine.py` 第 88-90 行
- **原因**: `_execute_task` 在 try 块末尾执行 `scheduler.get_job(f"task_{task_id}")`，当任务由手动触发（非调度器触发）时，可能未获取到 job 或 job 状态异常，导致 `next_run_time` 属性访问抛 `AttributeError`，进入 except 分支标记为 failed
- **修复建议**: 将 `next_run` 更新逻辑包裹在独立的 try-except 中，不影响任务结果状态
- **严重级别**: 🟡 中等（功能可用但状态不准确）

### BUG-003 [轻微] 定时任务创建后 next_run 为 null

- **现象**: 创建任务后，任务列表中"下次执行时间"显示为 `-`
- **位置**: `backend/app/routers/scheduler.py` `create_task` + `backend/app/scheduler/engine.py` `add_task_to_scheduler`
- **原因**: `add_task_to_scheduler` 中更新 `next_run` 的逻辑依赖 scheduler 已启动，但 TestClient 环境下 scheduler 可能未完全初始化。需确认生产环境是否正常。
- **修复建议**: 在 `create_task` 返回前主动计算 next_run 并写入
- **严重级别**: 🟢 轻微（展示问题）
