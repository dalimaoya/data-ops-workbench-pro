### BUG-010 CURRENT_VERSION 硬编码 — ⚠️ 部分修复

**commit**: `201a0c3`

**修复内容审查**:
1. 新增 `_read_version()` 函数，优先读 `version.txt` → 其次 `package.json` → 回退硬编码 ✅
2. `CURRENT_VERSION = _read_version()` 替代原硬编码 ✅

**问题**:
- 项目根目录下 **不存在 `version.txt` 和 `package.json`**，`_read_version()` 将始终走回退分支返回硬编码 `"3.10.0"`
- `build.sh` 中未添加生成 `version.txt` 的步骤，构建产物中也不会自动包含此文件
- 这意味着修复逻辑虽然正确，但在实际运行中仍等效于硬编码

**建议**: 在 `build.sh` 中增加 `echo "$VERSION" > version.txt` 步骤，或在项目根目录维护一个 `version.txt` 文件。

**结论**: 代码逻辑正确但缺少配套文件，实际效果为部分修复。

### BUG-011 build.sh 用 PyInstaller — ⚠️ 部分修复

**commit**: `1ccb7bd`

**修复内容审查**:
1. 依赖安装从 `pyinstaller` 改为 `nuitka ordered-set` ✅
2. 构建命令从 `pyinstaller app.spec` 改为 `python -m nuitka --standalone` ✅
3. 产物路径从 `dist/app` 改为 `dist/main.dist` ✅
4. 使用 `--enable-plugin=anti-bloat` 和 `--remove-output` 优化产物体积 ✅

**问题**:
- `--include-data-dir=web=web` 假设 backend 下有 `web` 目录作为静态资源，需确认该目录在构建环境中存在
- 未指定 `--output-filename` 参数，生成的可执行文件名为 `main.bin`（Linux）而非 `app`，下游启动脚本（`start.sh`）是否适配需验证
- 缺少 `--follow-imports` 等选项，如有动态导入的模块可能遗漏

**结论**: 主要修复到位（已从 PyInstaller 切换为 Nuitka standalone），但存在上述配套细节待确认，标记为部分修复。
