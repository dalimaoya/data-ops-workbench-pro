### 1. P1 Bug 修复验证

#### 1.1 sync-fields 合并策略

- **测试步骤**：
  1. 创建纳管表 demo_results（tc_id=3）
  2. 修改 remark 字段：field_alias="备注信息"、max_length=200、is_required=1
  3. 执行 `POST /api/table-config/3/sync-fields`
  4. 重新查询字段配置
- **结果**：sync 后 remark 字段 alias=备注信息、max_length=200、is_required=1 均保留
- **代码确认**：`_merge_fields()` 方法仅更新 `db_data_type`、`field_order_no`、`is_primary_key`、`is_system_field`、`sample_value`，不覆盖用户自定义的 `field_alias`、`max_length`、`is_required`、`is_editable`、`include_in_export`、`include_in_import`
- **状态**：✅ 通过

#### 1.2 sample_value 自动填充

- **测试步骤**：创建纳管表时检查 9 个字段的 sample_value
- **结果**：
  | 字段 | sample_value |
  |---|---|
  | id | 1 |
  | project_name | 项目A |
  | indicator_name | 销售额 |
  | indicator_value | 10000.50 |
  | report_date | 2026-01-15 |
  | status | published |
  | remark | 一季度数据 |
  | created_at | 2026-03-20 02:12:55 |
  | updated_at | 2026-03-20 02:12:55 |
- **代码确认**：`create_table_config()` 中调用 `fetch_sample_data()` 获取前 5 行，取每列第一个非 NULL 值写入 `sample_value`
- **状态**：✅ 通过

## 风险与建议

| # | 风险/建议 | 级别 | 说明 |
|---|---|---|---|
| 1 | 大数据量导出性能 | 低 | 当前全量加载到内存生成 Excel，万行级别以上可能需流式处理，但 MVP 可接受 |
| 2 | 并发回写竞态 | 低 | 当前无并发锁机制，多人同时回写同一表可能产生冲突，MVP 阶段风险可控 |
| 3 | 导出文件清理 | 建议 | exports 目录下文件无自动清理机制，建议后续增加过期清理 |
| 4 | operator_user 硬编码 | 建议 | 当前所有操作 operator_user="admin"，待 P3 登录/权限模块接入后替换为实际用户 |
