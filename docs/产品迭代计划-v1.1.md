## 一、v1.1 — 体验修正与基础完善

> 目标：修复现有体验问题，不增加新功能模块，保持稳定。

### 1.1 时间显示全局修正

**问题：** 页面上涉及时间的字段未统一使用北京时间格式显示。

| 排查范围 | 涉及页面 |
|---|---|
| 数据维护 - 更新时间 | 数据浏览页 |
| 版本回退 - 备份时间 | 备份版本列表、备份版本详情 |
| 日志中心 - 操作时间 | 系统操作日志、模板导出日志、模板导入日志、回写日志 |

**修正标准：**
- 所有时间统一显示为 `YYYY-MM-DD HH:mm:ss` 格式
- 时区统一为北京时间（UTC+8）
- 后端存储用 UTC，前端展示转北京时间；或后端直接存储/返回北京时间

### 1.2 结构状态中文化

**问题：** 部分页面结构状态显示英文原始值 `normal`、`changed` 等，普通用户看不懂。

**修正标准：**
- `normal` → 正常
- `changed` → 已变化
- `error` → 检查失败
- 全局排查所有显示结构状态的位置，确保无遗漏

### 1.3 README 与关于页面补充

|---|---|---|
| 1.3.1 | README 加入仓库链接 | GitHub: https://github.com/dalimaoya/data-ops-workbench / Gitee: https://gitee.com/dalimaoya/data-ops-workbench |
| 1.3.2 | 系统"关于"页面 | 显示：系统版本号、功能简介、使用手册入口、GitHub/Gitee 链接 |

## 二、v1.2 — 模板安全加固

> 目标：提升模板机制的安全性，防止用户误操作导致数据问题。

### 2.1 模板 Excel 保护机制

**需求来源：** 当前下载的模板没有做数据格式校验，用户可以随意修改列头、删除字段列。

| # | 保护项 | 说明 |
|---|---|---|
| 2.1.1 | 表头行保护 | 标头行锁定，不可修改、不可删除 |
| 2.1.2 | 列结构保护 | 导出几个字段就锁定几列，不允许删除列 |
| 2.1.3 | 主键列保护 | 主键列单元格设为只读（已有数据行的主键不可改） |
| 2.1.4 | 工作表保护 | 使用 Excel 工作表保护功能，仅允许编辑"可编辑字段"的数据单元格 |

**关于主键与新增数据的处理：**
- v1.2 阶段：主键列对已有数据行锁定只读
- 但模板设计必须为"新增行"预留扩展空间——真实上线版本必须支持用户在 Excel 中批量新增数据行再导入
- 具体方案：模板中已有数据区域主键锁定，模板底部预留空白行区域（不锁定主键列），用户可在此填写新增数据
- 新增行的完整回写（INSERT）能力在 v2.0 落地，但 v1.2 的模板结构设计要提前兼容，避免后续破坏性改动

### 2.2 导入校验增强

| # | 校验项 | 说明 |
|---|---|---|
| 2.2.1 | 列数校验 | 上传文件列数与模板定义不一致时阻断 |
| 2.2.2 | 列名校验 | 列头名称与模板定义不一致时阻断 |
| 2.2.3 | 主键不可变校验 | 检测主键值是否被修改，修改则报错 |

**优先级：P1 — v1.1 之后优先处理**

## 四、v1.4 — 独立打包部署

> 目标：实现零环境依赖的独立部署包，拷贝即运行。

### 4.1 PyInstaller 打包

| # | 事项 | 说明 |
|---|---|---|
| 4.1.1 | Linux 打包 | 打包为单文件 ELF 可执行文件 |
| 4.1.2 | Windows 打包 | 打包为单文件 .exe 可执行文件 |
| 4.1.3 | 前端构建产物内嵌 | 前端 build 产物打入包内，由后端静态托管 |
| 4.1.4 | 启动脚本适配 | `start.sh` / `start.bat` 直接调用打包产物，无需 Python 环境 |

### 4.2 发布包结构

```
data-ops-workbench-v1.4-win64/
├── start.bat
├── server/
│   └── app.exe          # 打包后的后端+前端
├── data/                 # 运行时数据（首次启动自动生成）
├── backups/
├── logs/
└── README.md
```

**最终效果：** 用户下载 zip → 解压 → 双击 start.bat → 浏览器访问。不需要装 Python、Node 或任何东西。

**优先级：P1 — 这是用户明确提出的刚需（纯内网环境）**
