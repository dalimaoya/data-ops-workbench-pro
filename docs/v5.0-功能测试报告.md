## 2. 试用期 + 插件授权

### TC-2.1 试用期状态查询
- **操作**：GET /api/plugins/trial-status (Bearer token)
- **预期**：返回试用期剩余天数
- **实际**：HTTP 500 Internal Server Error
- **结果**：**FAIL**
- **原因**：`TypeError: can't subtract offset-naive and offset-aware datetimes`。`TrialActivation.expires_at` 从 SQLite 读取后是 timezone-naive 的 datetime, 而 `_now_bjt()` 返回 timezone-aware (UTC+8)。执行 `(trial.expires_at - now).total_seconds()` 时类型不匹配。
- **风险等级**：**P1** — 前端试用期倒计时功能完全不可用
- **复现稳定性**：100% 可复现
- **代码位置**：`backend/app/routers/plugins.py:92`
- **建议修复**：在比较前将 `trial.expires_at` 转为 aware datetime, 或统一使用 naive datetime

### TC-2.2 插件加载列表
- **操作**：GET /api/plugins/loaded
- **预期**：7 个 builtin + 12 个 extension
- **实际**：Total=19, Builtin=7, Extension=12 -- 数量正确
- **结果**：**PASS**

### TC-2.3 builtin 插件始终 authorized=true
- **操作**：检查 /api/plugins/loaded 返回的 builtin 插件 authorized 字段
- **实际**：7 个 builtin 插件全部 authorized=true：plugin-backup, plugin-batch-ops, plugin-db-manager, plugin-health-check, plugin-notification, plugin-scheduler, plugin-sql-console
- **结果**：**PASS**

### TC-2.4 extension 插件试用期内 authorized=true
- **操作**：检查 /api/plugins/loaded 返回的 extension 插件 authorized 字段
- **实际**：数据库中存在有效的 trial_activation 记录（expires_at=2026-04-28），12 个 extension 插件全部 authorized=true
- **结果**：**PASS**

### TC-2.5 导航栏只显示已授权的扩展插件
- **操作**：源码审查 `frontend/src/layouts/MainLayout.tsx:184-189`
- **实际**：`buildMenuGroups` 函数通过 `loadedSet` 过滤，extension 插件必须 `loaded=true` 且 `authorized !== false` 才会出现在导航中

## 6. 更新检测

### TC-6.1 检查更新
- **操作**：GET /api/auth/check-update
- **实际**：`{"current_version":"0.0.0", "latest_version":null, "has_update":false, "error":"无法获取版本信息"}`
- **结果**：**FAIL**
- **原因**：`current_version` 显示为 `0.0.0` 而非 `4.5.0`。根源在 `backend/app/routers/auth.py:114`，版本文件路径通过 `__file__` 的相对父目录计算为 `backend/version.txt`，但实际文件位于项目根目录 `version.txt`。使用 uvicorn 以模块方式运行时路径正确，但当前测试环境下路径解析不一致。另外 Gitee API 返回失败可能是网络/仓库配置问题。
- **风险等级**：P2（功能受损但不阻塞核心业务）
