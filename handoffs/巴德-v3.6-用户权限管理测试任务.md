# 测试任务：v3.6 用户权限管理

## 阶段
迭代测试补测（第一批 P0 第 4 项）

## 背景
v3.6 新增三角色用户权限管理（commit 0753a6d），是权限安全基线功能。未经巴德正式测试。

## 测试范围
基于 CHANGELOG v3.6.0 功能点：

### 用户管理（管理员功能）
1. 用户列表：查看所有用户、角色、状态、最后登录时间
2. 新增用户：指定用户名、显示名、角色
3. 编辑用户：修改显示名和角色
4. 禁用/启用用户（保护最后一个管理员不可禁用）
5. 重置密码

### 个人设置
6. 修改密码（需验证旧密码）
7. 修改显示名
8. 导航栏显示当前用户名和角色

### 三角色权限控制
9. 管理员（admin）：全部功能 + 系统设置 + 用户管理
10. 操作员（operator）：数据浏览、导出、导入、回写、日志
11. 只读（viewer）：仅数据浏览和日志查看
12. 前端菜单按角色隔离
13. 后端 API 角色校验装饰器（403 拦截越权）
14. 操作日志记录实际操作人

### 测试要点
- [ ] 管理员可查看用户列表，字段完整
- [ ] 管理员可新增用户，三种角色均可创建
- [ ] 管理员可编辑用户显示名和角色
- [ ] 最后一个管理员不可被禁用
- [ ] 禁用用户后该用户无法登录
- [ ] 管理员可重置其他用户密码
- [ ] 用户可修改自己的密码（需验证旧密码）
- [ ] 用户可修改自己的显示名
- [ ] 操作员看不到数据源管理、用户管理、版本回退菜单
- [ ] 只读用户看不到上传/回写按钮
- [ ] 操作员/只读用户直接调用受限 API → 返回 403
- [ ] 操作日志中记录的是实际操作人（非硬编码 admin）
