## 三、必须对接的接口

### 1. 微信登录重定向
```http
GET https://auth.aiusing.net/api/auth/wechat/redirect?redirect_uri=<工作台回调地址>
```

### 2. token 在线校验
```http
GET https://auth.aiusing.net/api/auth/verify
Authorization: Bearer <JWT_TOKEN>
```

### 3. 插件授权校验
```http
GET https://auth.aiusing.net/api/license/check?plugin=<plugin_key>
Authorization: Bearer <JWT_TOKEN>
```

### 4. 公钥下载
```http
GET https://auth.aiusing.net/public/keys/jwt_public.pem
```

## 四、实现要求

### 登录闭环
- 未登录用户进入工作台时可看到统一认证登录入口
- 点击后能跳转到微信登录
- 认证平台回跳后能正确解析 token
- token 校验成功后进入工作台主页

### 授权闭环
- 至少为 1 个插件功能接入授权校验
- 未授权时必须有明确提示
- 不允许静默失败或无反馈

### 离线验签
- 工作台可下载并保存公钥
- 支持基于本地公钥的 RS256 验签
- 若在线 verify 与离线验签策略冲突，需在回传中写明当前实际实现策略

### 工程要求
- 不重做认证平台
- 不扩展后台管理功能
- 不擅自扩大为多账号/多组织体系
- 实现后补充必要配置说明
