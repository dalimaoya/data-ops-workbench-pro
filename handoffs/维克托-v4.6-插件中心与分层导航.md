### 新建 plugin_status 表
```sql
CREATE TABLE plugin_status (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    plugin_id VARCHAR(100) UNIQUE NOT NULL,  -- manifest 中的 id
    enabled BOOLEAN DEFAULT 0,
    enabled_by VARCHAR(100),
    enabled_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 后端接口
- `GET /api/plugins/all` — 返回所有插件（含 layer/category/enabled 状态）
- `PUT /api/plugins/{plugin_id}/toggle` — 启用/停用扩展插件（仅 admin）
- 修改现有 `GET /api/plugins/loaded` — 只返回 builtin + 已启用的 extension

## 任务 2：插件中心页面（1.5 天）

### 前端新页面 `/plugin-center`

**页面结构：**
- 顶部：标题"🧩 插件中心" + 搜索框
- 分类筛选标签：全部 | AI 智能 | 数据治理 | 报表审计 | 流程协作 | 外部集成 | 资源市场
- 插件卡片网格（每行 3-4 个）

**每张卡片：**
- 图标（可用 emoji 或 Ant Design 图标）
- 插件名称（display_name）
- 一句话描述（description）
- 分类标签（category 对应的中文）
- 状态开关：已启用（绿色 Switch）/ 未启用（灰色 Switch）
- 预留：锁图标状态（`"license": "required"` 时显示，当前全部为 `"free"`）

**交互：**
- 点击 Switch 切换启用/停用，调用 `PUT /api/plugins/{id}/toggle`
- 启用后导航栏实时出现对应菜单
- 停用后导航栏实时移除
- 仅 admin 可操作 Switch，其他角色只可查看

**分类映射：**
| category | 中文 | 图标 |
|----------|------|------|
| ai | AI 智能 | 🤖 |
| governance | 数据治理 | 📊 |
| report | 报表审计 | 📋 |
| workflow | 流程协作 | ✅ |
| integration | 外部集成 | 🔗 |
| market | 资源市场 | 🏪 |

## 任务 3：导航栏分组渲染（1 天）

### 导航结构调整

```
── 底座 ──
📊 工作台总览
📁 数据源管理
📋 纳管表配置
🔍 数据浏览
📥 数据维护
── 基础工具 ──
🗄️ 库表管理
💻 SQL 控制台
⏰ 定时任务
🏥 健康巡检
📦 备份迁移
⚡ 批量操作
── 扩展功能 ──（动态：仅显示已启用的扩展插件）
🤖 AI 智能助手
...（按启用顺序）
── 系统 ──
📝 日志中心
⏪ 版本回退
👥 用户管理
⚙️ 系统设置
🧩 插件中心
```

### 分组分隔
- 每组之间用分割线 + 灰色小标题（"基础工具"/"扩展功能"/"系统"）
- 扩展功能组：无启用插件时不显示该组
- 通知中心（plugin_notification）不在侧边栏，改为顶部导航栏铃铛图标

### 技术要点
- 前端菜单渲染从 `GET /api/plugins/loaded` 获取插件列表
- 按 layer 字段分组排列
- 底座菜单硬编码（不从插件接口获取）
- builtin 和已启用的 extension 动态渲染
