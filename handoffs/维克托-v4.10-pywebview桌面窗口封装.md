# 维克托 — v4.10 pywebview 桌面窗口封装

> 派发人：斯维因 | 日期：2026-03-27 | 优先级：P1
> 版本定位：接入统一认证平台（REQ-007）前的最后一个独立版本收尾

---

## 背景与目标

当前 v4.9 的用户体验是：GUI 启动器（tkinter 状态窗口）+ 自动打开系统浏览器。
用户感知是「一个服务 + 一个浏览器 Tab」，不够像成熟桌面产品。

目标：用 **pywebview** 将 Web UI 内嵌为原生桌面窗口，同时保留浏览器访问入口，
让用户感知变为「一个桌面应用」。

---

## 技术方案

- **核心库：** `pywebview`（调用 Win10/11 自带 WebView2，无需额外 Chromium）
- **兼容降级：** `pywebview` 不可用时自动回退现有 `webbrowser.open()` 行为
- **编译：** Nuitka 编译时需追加 `pywebview` 相关依赖声明

---

## 实现要求

### 1. 启动流程改造

原流程：服务就绪 → `webbrowser.open(f"{URL}/loading")`

新流程：
```
服务就绪
  ├─ HAS_WEBVIEW=True  → 打开 pywebview 桌面窗口（加载 URL）
  └─ HAS_WEBVIEW=False → fallback：webbrowser.open(URL)（保持原行为）
```

### 2. pywebview 窗口配置

```python
import webview  # pywebview

window = webview.create_window(
    title=f"数据运维工作台  v{CURRENT_VERSION}",
    url=URL,
    width=1440,
    height=900,
    min_size=(1024, 600),
    resizable=True,
)
webview.start()  # 阻塞，窗口关闭后继续
```

**要求：**
- 窗口标题与当前版本号保持一致
- 默认尺寸 1440×900，最小尺寸 1024×600
- 窗口可最大化/最小化/调整大小
- 无需自定义无边框标题栏（保持系统默认）

### 3. 保留浏览器访问入口

以下入口必须保留，不得移除：
- tkinter 状态窗口内「打开浏览器」按钮 → `webbrowser.open(URL)`
- 系统托盘右键菜单「打开浏览器」→ `webbrowser.open(URL)`
- 新增：pywebview 窗口内右键菜单或标题栏菜单 → 「在浏览器中打开」

### 4. 线程安全

- `webview.start()` 必须在主线程调用（pywebview 要求）
- tkinter 状态窗口与 pywebview 窗口不能同时作为主线程窗口
- 推荐方案：服务就绪后，tkinter 窗口隐藏（`root.withdraw()`），主线程切换到 `webview.start()`；pywebview 窗口关闭后，tkinter 窗口重新显示

### 5. 可选依赖处理

```python
try:
    import webview
    HAS_WEBVIEW = True
except ImportError:
    HAS_WEBVIEW = False
```

`HAS_WEBVIEW=False` 时行为与 v4.9 完全一致，不报错、不影响核心功能。

### 6. 打包配置更新

- `build.bat` / `build.sh`：确认 pywebview 及其 WebView2 loader 被正确打包进 Nuitka 产物
- `setup.iss`：版本号更新为 v4.10.0
- `version.txt`：更新为 4.10.0
- `requirements.txt`（如有）：新增 `pywebview>=4.0`
