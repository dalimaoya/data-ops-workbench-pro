## 任务 1：启动 Loading 页（1 天）

### 目标
用户双击启动后，浏览器打开的不是白屏等待，而是带品牌感的 loading 页面，显示启动进度百分比。

### 后端
新增 `GET /api/startup-progress` 接口：
- 返回当前启动阶段和进度百分比
- 阶段定义：
  - `initializing_db`（0-20%）：数据库初始化
  - `loading_plugins`（20-50%）：加载插件
  - `starting_server`（50-80%）：启动服务组件
  - `ready`（100%）：就绪
- 此接口在服务尚未完全就绪时也要能响应（注册在 lifespan 之前）

### 前端 Loading 页
- 路由：`/loading`（独立于主布局，不需要登录）
- 页面设计：
  - 居中布局，渐变背景（和登录页风格统一）
  - 产品 logo + 产品名"数据运维工作台"
  - 进度条（Ant Design Progress 组件）
  - 当前阶段文字："正在初始化数据库..." / "正在加载插件..." / "正在启动服务..." / "启动完成"
  - 进度百分比数字
- 行为：
  - 每秒轮询 `/api/startup-progress`
  - 进度 100% 后延迟 0.5 秒，自动跳转登录页
  - 如果直接访问时服务已就绪，立即跳转登录页

### 启动脚本配合
- start.sh / start.bat 中，浏览器打开的 URL 从 `http://localhost:8580` 改为 `http://localhost:8580/loading`
- launcher.py 同步修改

## 任务 2：离线状态提示（0.5 天）

### 目标
明确告知用户哪些功能需要联网，离线时不报错而是友好提示。

### 后端
新增 `GET /api/system/network-status` 接口：
- 检测是否能访问外网（尝试连接已配置的 AI 平台 API 端点，或 fallback 到 ping 公共 DNS）
- 返回 `{ "online": true/false, "checked_at": "..." }`
- 缓存结果 60 秒，避免频繁检测

### 前端
- AI 功能入口（所有 AI 按钮）在离线时：
  - 按钮不禁用，但点击后弹出提示："AI 功能需要联网使用，请检查网络连接"
  - 或者如果 AI 设置页未配置任何模型平台，提示："请先在系统设置中配置 AI 模型"
- 系统设置页新增"网络状态"显示：
  - 在线：绿色圆点 + "网络正常"
  - 离线：红色圆点 + "网络未连接，AI 功能不可用"
