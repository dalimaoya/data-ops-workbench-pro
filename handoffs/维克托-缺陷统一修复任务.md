## P0 — 阻塞性缺陷（必须修复）

### BUG-001 [v3.10] 导入校验未读取 DELETE 标记
- **位置**：`data_maintenance.py` → `import_template()`
- **现象**：`operation_col_idx` 定义后从未使用，导入校验完全忽略 `_操作` 列的 DELETE 标记
- **预期**：导入时识别 `_操作` 列值为"删除"/"DELETE"的行，标记为待删除
- **测试报告**：`docs/v3.10-测试报告.md`

### BUG-002 [v3.10] 回写缺少 DELETE 执行分支
- **位置**：`data_maintenance.py` → `writeback()`
- **现象**：只有 UPDATE 和 INSERT 分支，没有 DELETE 逻辑
- **预期**：对标记为删除的行，先备份再执行 DELETE，结果页显示删除数
- **测试报告**：`docs/v3.10-测试报告.md`

## P1 — 重要缺陷（应修复）

### BUG-003 [v3.10] 前端差异预览缺少 delete 类型
- **位置**：前端 `DiffPreview.tsx` → change_type 映射
- **现象**：缺少 `delete` 类型，无红色标签
- **预期**：删除行在差异预览中显示红色标签
- **测试报告**：`docs/v3.10-测试报告.md`

### BUG-004 [v3.10] 回写结果页不展示删除行数
- **位置**：前端回写结果页 + 统计卡片
- **现象**：只展示更新数和新增数，没有删除数
- **预期**：结果页显示更新数 + 新增数 + 删除数
- **测试报告**：`docs/v3.10-测试报告.md`

### BUG-005 [v3.9] browse_table_data 缺少 allow_insert_rows
- **位置**：`data_maintenance.py` → `browse_table_data` 接口
- **现象**：接口返回值缺少 `allow_insert_rows` 字段
- **预期**：前端根据此字段决定是否显示新增按钮
- **测试报告**：`docs/v3.9-测试报告.md`

### BUG-006 [v3.5] 主键不可变校验错误提示硬编码
- **位置**：`data_maintenance.py` → 主键校验逻辑
- **现象**：错误提示中原始值为硬编码占位符 `"(原始值)"`，未传入数据库真实主键值
- **预期**：提示包含数据库中的真实原始主键值
- **测试报告**：`docs/v3.5-测试报告.md`

## P2 — 低风险缺陷（建议修复）

### BUG-007 [v3.9] INSERT 模板预留行数硬编码 50 行
- **位置**：模板导出逻辑
- **现象**：空白行固定 50 行，无配置入口
- **修复方案**：参见 `briefs/INSERT行数限制补丁需求.md`，默认 200 行 + 纳管表可配置
- **测试报告**：`docs/v3.9-测试报告.md`

### BUG-008 [v3.6] 三个只读接口缺少角色校验
- **位置**：`get_import_diff`、`get_diff_report`、`get_export_info`
- **现象**：未加 `require_role` 装饰器，viewer 可直接调 API 访问
- **预期**：加上角色校验或确认为设计意图
- **测试报告**：`docs/v3.6-测试报告.md`

### BUG-009 [v3.8] launcher.py 版本比较逻辑错误
- **位置**：`launcher.py` → 版本更新检查
- **现象**：用字符串 `!=` 比较版本号，降级场景也会弹出"发现新版本"
- **预期**：使用语义化版本比较，仅当远程版本大于本地版本时提示
- **测试报告**：`docs/v3.4-v3.8-构建部署测试报告.md`

### BUG-010 [v3.8] CURRENT_VERSION 硬编码
- **位置**：`launcher.py`
- **现象**：每次发版需手动更新版本号
- **预期**：从构建产物或配置文件自动读取
- **测试报告**：`docs/v3.4-v3.8-构建部署测试报告.md`

### BUG-011 [v3.4] build.sh 与 CI 构建方式不一致
- **位置**：`build.sh`
- **现象**：本地 build.sh 用 PyInstaller，CI 已迁移 Nuitka
- **预期**：统一为 Nuitka，或 build.sh 标注为开发用途
- **测试报告**：`docs/v3.4-v3.8-构建部署测试报告.md`
