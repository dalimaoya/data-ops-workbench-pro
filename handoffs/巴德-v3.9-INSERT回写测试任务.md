# 测试任务：v3.9 INSERT 回写功能

## 阶段
迭代测试补测（第一批 P0 第 2 项）

## 背景
v3.9 新增 INSERT 回写支持（commit d9e12ce），允许用户在模板底部空白区填写新数据，导入后回写执行 INSERT。此版本未经巴德正式测试。

## 测试范围
基于 CHANGELOG v3.9.0 功能点：

### 功能点
1. 模板底部空白区域支持填写新数据
2. 导入自动识别新增行（主键不在数据库中）
3. 差异预览绿色高亮新增行
4. 回写执行 INSERT，结果页显示更新数+新增数

### 测试要点
- [ ] 模板底部空白区是否可正常填写新数据
- [ ] 新增行填写后导入，校验是否正确识别为新增行
- [ ] 新增行主键不在数据库中 → 识别为 INSERT
- [ ] 新增行主键已在数据库中 → 应识别为 UPDATE（非新增）
- [ ] 差异预览：新增行绿色高亮，与更新行（蓝色）区分
- [ ] 回写执行 INSERT 后，数据确实写入业务表
- [ ] 结果页正确显示更新数和新增数
- [ ] 混合场景：同一批次含更新行 + 新增行
- [ ] 边界：新增行缺少必填字段 → 校验报错
- [ ] 边界：新增行主键为空 → 校验报错
- [ ] 边界：新增行超过模板预留空白行数（当前 50 行）时的表现
- [ ] 权限：只读用户上传含新增行的模板 → 应被拒绝
