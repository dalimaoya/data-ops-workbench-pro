## 十、缺陷清单

### 缺陷 1：认证平台 TLS 证书不受信，统一认证主链路整体阻断
- 严重级别：**P1 阻塞**
- 现象：
  - `/api/auth/wechat/redirect-url` 返回 500
  - `/api/auth/public-key/refresh` 返回 500
  - `/api/auth/verify` 返回 500
  - `/api/license/check` 返回 500
- 证据：服务日志统一报错
  - `SSL: CERTIFICATE_VERIFY_FAILED`
  - `self-signed certificate`
- 影响：
  - 登录入口无法发起
  - verify 无法在线校验
  - license 无法在线授权校验
  - 公钥无法下载，离线验签首次也无法建立
- 建议处理：
  1. 优先修复 `auth.aiusing.net` 的正式证书与证书链
  2. 修复后重新执行完整联调
  3. 不建议以关闭校验作为发布口径替代正式修复

### 缺陷 2：统一认证相关远端异常未被业务化处理，前端/接口直接暴露 500
- 严重级别：**P1 阻塞**
- 现象：多个统一认证接口在上游 TLS/网络异常时直接返回 `500 Internal Server Error`
- 预期：根据产品/交互说明，应给出明确提示，例如：
  - `认证服务暂时不可用，请稍后重试`
  - `公钥获取失败，暂无法完成本地校验`
- 影响：
  - 用户端无法获得明确可执行提示
  - QA 无法按产品口径确认异常链路达标
- 建议处理：
  1. 在统一认证服务层补齐 `httpx.ConnectError` / TLS 异常捕获
  2. 统一转换为 `HTTPException(503/502)` + 清晰中文文案
  3. 前端登录页/回调页/授权页显示与产品方案一致的提示

### 缺陷 3：离线验签首次场景对公钥缺失依赖在线下载，当前失败后未降级为明确离线不可用提示
- 严重级别：**P2 主要缺陷**
- 现象：携带伪造 token 调用 `/api/auth/offline-verify` 时，由于本地无公钥缓存，接口尝试在线下载公钥；下载失败后直接 500
- 影响：
  - 首次使用时离线验签不可达
  - “在线失败→离线兜底”在无缓存场景下实际不可用
- 建议处理：
  1. 区分“无缓存 + 下载失败”与“签名校验失败”
  2. 返回明确提示：`公钥获取失败，暂无法完成本地校验`
  3. 在 UI 侧清晰提示离线校验不可用，而非 500
