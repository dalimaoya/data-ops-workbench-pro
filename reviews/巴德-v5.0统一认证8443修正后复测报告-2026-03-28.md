## 三、8443 入口验证结果

### 1. 代码口径复核结果
复核结果：**通过**

证据：
1. `backend/app/services/unified_auth.py`
   - `AUTH_PLATFORM_BASE_URL` 默认值已为：`https://auth.aiusing.net:8443`
   - `AUTH_PUBLIC_KEY_URL` 由该 base URL 拼出：`https://auth.aiusing.net:8443/public/keys/jwt_public.pem`
2. 同文件中以下链路均基于同一个 `AUTH_PLATFORM_BASE_URL`：
   - `build_wechat_redirect_url()`
   - `verify_token_online()`
   - `check_plugin_license()`
   - `download_public_key()`
3. `frontend/src/pages/Login.tsx`
   - 登录页展示文案已明确标注“当前联调：8443”
   - 页面展示认证来源：`https://auth.aiusing.net:8443`
4. `backend/app/routers/auth.py`
   - `/api/auth/platform-config` 会返回当前后端使用的 `base_url`
   - `/api/auth/wechat/redirect-url`、`/api/auth/verify`、`/api/auth/offline-verify`、`/api/auth/public-key/refresh` 已完整接线
5. `backend/app/routers/license.py`
   - `/api/license/check` 已接线到统一认证服务层

### 2. 远端 8443 入口实测结果
复核结果：**通过**

实测现象：
1. 访问 `https://auth.aiusing.net:8443/public/keys/jwt_public.pem` 返回 **HTTP 200**，可直接取到 PEM 公钥头：
   - `-----BEGIN PUBLIC KEY-----`
2. 对比访问 `https://auth.aiusing.net/public/keys/jwt_public.pem`（未带 8443）时，当前环境返回证书校验失败，报错为：
   - `self-signed certificate`

判断：
- 当前远端统一认证有效联调入口确认为 **8443**，与正式背景结论一致
- 443 口径不应继续作为本次联调入口使用

## 四、登录链路结果

### 1. 登录入口代码检查
结果：**通过（代码层）**

检查点：
- 登录页已切换为统一认证页
- 点击“微信登录”调用前端 `fetchWechatRedirectUrl()`
- 前端回调页 `/auth/callback` 会：
  1. 从 URL fragment 提取 `token/access_token`
  2. 先尝试 `refreshPublicKey()`
  3. 再优先 `verifyTokenOnline(token)`
  4. 在线失败后降级 `verifyTokenOffline(token)`
  5. 建立统一认证会话并落本地
- `AuthContext` 启动时会优先在线 verify，失败后再尝试离线验签恢复登录态

### 2. 8443 登录入口远端实测
结果：**部分通过**

实测项：
1. 请求 `https://auth.aiusing.net:8443/api/auth/wechat/redirect?redirect_uri=http://127.0.0.1:18080/auth/callback`
2. 远端返回 **HTTP 400**：
   - `{"message":"redirect_uri not allowed","error":"Bad Request","statusCode":400}`

判断：
- 该结果至少说明：**工作台应命中的微信重定向入口已经可以在 8443 上响应，链路没有再走错到 443**
- 但由于本轮使用的是临时本地 callback 地址，不在远端允许名单中，**无法完成微信登录成功回跳闭环**

### 3. 登录成功闭环结论
结果：**未完成，受环境限制**

未完成项：
- 微信授权成功回跳
- 回调页拿到真实 token
- 在线 verify 成功建立在线登录态
- 登录后进入首页
- 刷新后恢复登录态

阻塞原因：
- 无有效测试回调域名 / 无可交互微信授权环境 / 无有效测试 token

## 五、授权链路结果

### 1. verify 链路
结果：**部分通过**

实测项：
- 请求：`https://auth.aiusing.net:8443/api/auth/verify`
- 携带假 token：`Bearer fake.invalid.token`
- 返回：**HTTP 401**
- 返回体：`Invalid or expired token`

判断：
- 说明 verify 远端接口在 8443 上**可达且已正确响应鉴权失败口径**
- 但因无有效 token，**无法证明 verify 成功闭环**

### 2. 插件授权 license check 链路
结果：**部分通过**

实测项：
- 请求：`https://auth.aiusing.net:8443/api/license/check?plugin=plugin_ai_assistant`
- 携带假 token：`Bearer fake.invalid.token`
- 返回：**HTTP 401**
- 返回体：`Invalid or expired token`

代码补充检查：
- 后端 `check_plugin_license()` 会将工作台插件名 `plugin-ai-assistant` 转为远端口径 `plugin_ai_assistant`
- 前端 `PluginGuard` 已接入 `/api/license/check`
- `request.ts` 会自动把本地 token 挂到请求头，授权失败时遇到 401 会清理登录态
- `PluginGuard` 对未授权返回展示 403 拦截页，对离线缓存场景展示 warning 提示

判断：
- 说明 8443 上的授权校验链路**可达**，且工作台插件 key 映射逻辑仍在
- 但本轮**无法验证已授权账号放行、未授权账号 403 拦截、离线授权缓存提示是否实际符合预期**，因为缺少真实账号样本

### 3. 授权链路结论
结果：**只完成“链路可达与异常口径”复测，未完成“正向授权/未授权拦截”业务闭环复测**

## 六、离线验签结果

### 1. 公钥下载与缓存
结果：**部分通过**

实测项：
- `https://auth.aiusing.net:8443/public/keys/jwt_public.pem` 可直接返回 200
- 返回内容为有效 PEM 头：`BEGIN PUBLIC KEY`

代码检查：
- 后端 `download_public_key()` 会把公钥缓存到：`data/auth/jwt_public.pem`
- `/api/auth/public-key/refresh` 已提供强制刷新入口
- 前端回调页会在在线校验前先尝试刷新公钥

判断：
- 公钥下载入口本身**可用**
- 工作台代码中**已具备本地缓存设计**

### 2. 离线验签实现检查
结果：**代码层通过，成功闭环未实证**

代码检查结论：
- `verify_token_offline()` 使用 `python-jose` 按 `RS256` 做本地验签
- 校验逻辑包含：
  - 优先读本地缓存公钥
  - 缓存不存在时尝试拉取公钥
  - 验签失败返回 401
  - 从 payload 中提取 `account_id/sub/openid/user_id`
  - 输出 `mode=offline`

### 3. 离线验签实测结果
结果：**仅完成失败口径确认，未完成成功验签**

已确认：
- 代码中离线验签失败会走 401/重新登录口径，不存在静默通过逻辑

未确认：
- 使用真实、未过期、由统一认证平台签发的 RS256 token 完成离线验签成功
- 在线 verify 不可用时，工作台是否能稳定进入离线受控态

阻塞原因：
- 本轮无真实有效 token，无法做正向 RS256 离线验签成功实测

### 4. 离线验签结论
结果：**公钥下载链路已证实可用；离线验签代码路径存在且失败口径合理；但离线验签成功闭环尚未完成实测**
